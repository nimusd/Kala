Spectrum Blender Documentation

Overview

The Spectrum Blender is a spectral morphing tool that performs per-harmonic linear interpolation between two input spectra. It takes two spectrum inputs (A and B) and a position control, producing a smooth blend at every harmonic.

This is the modular building block for timbral morphing - drive the position with an Envelope Engine or LFO to morph between timbres over a note’s lifetime.

Category

Shaping (Orange) - Transforms spectral character

Core Concept

output[h] = spectrumA[h] × (1 - position) + spectrumB[h] × position

Where h is the harmonic index and position ranges from 0.0 (pure A) to 1.0 (pure B). Every harmonic is interpolated independently, so the blend is smooth across the entire spectrum.

Ports

Input Ports

Port Data Type Default Description
spectrumA Spectrum (empty) First spectrum source (position = 0.0)
spectrumB Spectrum (empty) Second spectrum source (position = 1.0)
position Control 0.5 Blend position between A and B

Output Ports

Port Data Type Description
spectrumOut Spectrum Blended harmonic amplitudes

Parameters

Parameter Type Default Range Description
position Float 0.5 0.0-1.0 Static blend position (overridden when modulated)

Position Values

Position Result Use Case
0.0 Pure spectrum A Full first timbre
0.25 75% A, 25% B Subtle blend toward B
0.5 Equal mix Halfway morph
0.75 25% A, 75% B Mostly second timbre
1.0 Pure spectrum B Full second timbre

Mismatched Spectrum Sizes

When spectra A and B have different numbers of harmonics, the output size equals the larger of the two. Harmonics beyond the range of the smaller spectrum are treated as 0.0 amplitude:

spectrumA: 32 harmonics    [a0, a1, ..., a31]
spectrumB: 64 harmonics    [b0, b1, ..., b63]
output:    64 harmonics

For h = 0..31:  output[h] = a[h] × (1-pos) + b[h] × pos
For h = 32..63: output[h] = 0.0  × (1-pos) + b[h] × pos = b[h] × pos

This means blending toward a richer spectrum gradually fades in its extra harmonics.

Usage Patterns

Basic Timbre Blend

Static 50/50 mix of two timbres:

Harmonic Generator 1 ──→ spectrumA ──┐
                                      ├──→ Spectrum Blender ──→ Spectrum to Signal
Harmonic Generator 2 ──→ spectrumB ──┘

Envelope-Driven Morph

Morph between timbres over the note’s duration:

Harmonic Generator 1 ──→ spectrumA ──┐
                                      ├──→ Spectrum Blender ──→ Spectrum to Signal
Harmonic Generator 2 ──→ spectrumB ──┘
                                 ↑
Envelope Engine ──→ position

Set the envelope to ramp from 0.0 to 1.0 and the timbre evolves from A to B across the note.

LFO-Driven Oscillation

Continuously sweep between two timbres:

Harmonic Generator 1 ──→ spectrumA ──┐
                                      ├──→ Spectrum Blender ──→ Spectrum to Signal
Harmonic Generator 2 ──→ spectrumB ──┘
                                 ↑
LFO ──→ position

A slow LFO (0.5-2 Hz) creates a pulsing timbral effect. The position is clamped to 0.0-1.0 so bipolar LFO output is safe.

Multi-Stage Morphing (A → B → C)

Chain two blenders for three-way morphing:

HG 1 ──→ spectrumA ──┐
                       ├──→ Blender 1 ──→ spectrumA ──┐
HG 2 ──→ spectrumB ──┘                                 ├──→ Blender 2 ──→ Spectrum to Signal
                                                        │
HG 3 ────────────────────────────→ spectrumB ──────────┘

Drive Blender 1’s position through the first half of the note, then Blender 2’s position through the second half, for an A → B → C evolution.

With Rolloff Shaping

Shape the blended result before conversion:

HG 1 ──→ spectrumA ──┐
                       ├──→ Spectrum Blender ──→ Rolloff Processor ──→ Spectrum to Signal
HG 2 ──→ spectrumB ──┘

Shaped Inputs

Shape each source independently before blending:

HG 1 ──→ Rolloff 1 ──→ spectrumA ──┐
                                     ├──→ Spectrum Blender ──→ Spectrum to Signal
HG 2 ──→ Rolloff 2 ──→ spectrumB ──┘

This allows morphing between two fully-shaped timbres rather than raw harmonic content.

Important Behaviors

Linear Interpolation

The blend is strictly linear per harmonic. This means: - No harmonic generation or cancellation beyond what the math produces - Predictable, smooth transitions - CPU cost is minimal (one multiply-add per harmonic)

Position Clamping

When modulated via connections, the position is clamped to 0.0-1.0 after the connection function is applied. Values outside this range are safely bounded.

No State

The Spectrum Blender is stateless - it has no memory between samples. The output depends only on the current inputs. This means: - No reset needed at note boundaries - No artifacts from abrupt position changes - Deterministic output for given inputs

Empty Spectrum Handling

If a spectrum input has no connection, it defaults to an empty spectrum (all zeros). Blending with an empty spectrum effectively fades the other spectrum in or out based on position.

Comparison with Other Shapers

Container Domain Effect
Rolloff Processor Spectrum Power-law harmonic attenuation
Spectrum Blender Spectrum Linear interpolation between two spectra
Formant Body Signal Resonant filtering at formant frequencies
Noise Color Filter Signal Frequency-dependent noise shaping

The Spectrum Blender works in the spectrum domain (before conversion to audio), alongside the Rolloff Processor. While Rolloff shapes a single spectrum’s brightness curve, the Blender interpolates between two complete spectra.

Technical Notes

Interpolation Formula

For each harmonic h in the output:

output[h] = A[h] × (1 - position) + B[h] × position

This is equivalent to:

output[h] = A[h] + position × (B[h] - A[h])

Both forms produce identical results. The implementation uses the first form for clarity.

Output Size

The output spectrum size is max(A.numHarmonics, B.numHarmonics). This ensures no harmonic information is lost from either source.

CPU Cost

Processing cost is proportional to the output spectrum size: - One subtraction, two multiplications, and one addition per harmonic - Typically 64 harmonics = 256 floating-point operations per sample - Negligible compared to Spectrum to Signal’s sine calculations
