Karplus Strong Documentation

Overview

The Karplus Strong container is an extended physical string model (EKS framework, Jaffe & Smith 1983) that generates plucked string sounds with fine control over the excitation character, loop filter brightness, pick direction, and body resonance. It operates in two modes: String mode where the K-S model produces the entire sound, or Attack mode where it adds a pluck transient to an input signal.

Four excitation types (Noise, Soft Pluck, Blend, Finger Pluck) model different ways of setting a string in motion, while the EKS loop filter’s brightness and damping parameters independently control the tonal character and decay rate of the resulting string vibration. A body resonance biquad adds instrument-body coloring post-output.

Legato note transitions preserve the string’s vibrating state — the string continues ringing from the previous note rather than resetting.

Operating Modes

String Mode (mode = 0)

The K-S model generates the entire sound. signalIn is ignored. The output is a plucked string that decays naturally based on attackPortion and damping. After the attack phase completes, the output returns silence.

  • Suitable as a standalone pitched generator
  • Connect directly as the final output container
  • attackPortion controls how long the sound lasts relative to a full K-S decay

Attack Mode (mode = 1)

The K-S pluck acts as a transient that blends with an input signal. The pluck fades out while the input signal fades in, creating a natural pluck onset followed by sustained body.

burstWeight = (1.0 - progress) × mix
inputWeight = 1.0 - burstWeight
output = ksOutput × burstWeight + inputSignal × inputWeight

A cosine fade runs over the final 10% of the attack phase to prevent clicks at the transition point. After the attack phase completes, pure input signal passes through unmodified.

EKS Loop Filter

The core of the string model uses the Extended Karplus-Strong (EKS) loop filter — a first-order lowpass filter in the feedback loop, scaled by a feedback gain:

S   = 0.5 + brightness × 0.49         [range: 0.50 – 0.99]
rho = 0.990 + (1.0 − damping) × 0.0099 [range: 0.990 – 0.9999]

filtered = ((1.0 − S) × currentSample + S × prevSample) × rho

S is the stretching factor — it controls how much of the previous sample is blended in. High S means more previous sample contribution, which means less high-frequency energy loss per cycle (brighter, longer high-frequency sustain). Low S means each cycle loses high frequencies faster (darker, quicker brightness decay).

rho is the feedback gain — it controls frequency-independent energy loss per cycle. Lower rho (higher damping) produces faster overall decay.

Brightness vs Damping

Parameter Controls Effect at 0 Effect at 1
damping Feedback gain rho Very long sustain (rho ≈ 0.9999) Fast decay (rho ≈ 0.990)
brightness Loop filter S Dark (S ≈ 0.50, fast HF loss) Bright (S ≈ 0.99, slow HF loss)

These two parameters are independent: damping controls how fast the string dies away overall; brightness controls whether it dies away dark (fundamentals linger longer) or bright (upper harmonics linger longer).

Excitation Types

The excitation is the initial waveform loaded into the delay line when the string is plucked. Different excitation shapes produce different timbral characters.

0: Noise (default)

Filtered white noise. The pluckPosition parameter controls the noise filter cutoff:

cutoffHz = 200 + (1 − pluckPosition) × 8000
  • Bridge (pluckPosition = 0.0): Cutoff ≈ 8200 Hz — bright, full-bandwidth noise → metallic, sharp attack
  • Middle (pluckPosition = 0.5): Cutoff ≈ 4100 Hz — warm noise → balanced
  • Neck (pluckPosition = 1.0): Cutoff ≈ 200 Hz — heavily filtered noise → mellow, dark attack

blendRatio has no effect in pure Noise mode (it is used only in Blend mode).

Character: Classic Karplus-Strong. Rich harmonic content, position-dependent brightness.

1: Soft Pluck

A raised cosine hump centered in the delay buffer. pluckHardness controls the width of the hump:

widthSamples = N × (0.1 + 0.9 × (1 − pluckHardness))
  • pluckHardness = 0.0 (default): Full-width hump (width = N) — smooth, soft attack. Like a thumb or mallet
  • pluckHardness = 1.0: Narrow hump (width = 0.1×N) — pointed, brighter attack. Like a pick

The hump is DC-removed and normalized to ±1.0 before entering the delay line.

Character: Clean, smooth, bell-like or mallet-like. Less high-frequency content than Noise. Well-suited for marimba, vibraphone, or gentle acoustic plucks.

2: Blend

A linear mix of Noise and Soft Pluck. blendRatio controls the ratio:

delayLine[i] = (1 − blendRatio) × noise[i] + blendRatio × softPluck[i]
  • blendRatio = 0.0: All noise (same as Noise mode)
  • blendRatio = 0.5: Equal mix — textured but controlled
  • blendRatio = 1.0: All soft pluck (same as Soft Pluck mode)

pluckHardness controls the soft pluck component’s width, and pluckPosition controls the noise component’s filter cutoff. The blend is re-normalized after mixing.

Character: Tunable between gritty and smooth. The most versatile excitation type.

3: Finger Pluck

A rounded triangle waveform modelling the physical string displacement when a finger plucks and releases. pluckHardness controls both width and apex sharpness:

  • Width: 0 = wide (60% of delay length, like a thick thumb) → 1 = narrow (8%, like a nail or hard pick)
  • Rounding: 0 = heavily rounded apex (flesh-like softness) → 1 = sharp apex (hard surface)

The rounding is implemented as cosine smoothing at both the apex and the base of the triangle, producing a realistic string-displacement profile.

Character: Warm and organic at low hardness (nylon string, fingertip), bright and precise at high hardness (nail, plectrum). The most physically accurate excitation shape.

Pick Direction

After excitation generation, an EKS pick direction comb filter is applied to the delay line:

H(z) = 1 + p × z⁻¹
p = (pickDirection − 0.5) × 1.8   [range: −0.9 to +0.9]
pickDirection p Effect
0.0 −0.9 Notch at DC, boost at Nyquist → brighter, plectrum-like
0.5 (default) 0.0 Neutral — comb filter bypassed
1.0 +0.9 Notch at Nyquist, boost at DC → warmer, flesh-like

This models how different pick angles excite different harmonics at the moment of contact. The excitation is re-normalized after pick direction filtering to preserve level.

Body Resonance

An optional biquad bandpass resonator (fixed Q = 5.0) is applied to the K-S output post-feedback-loop (not in the string loop itself):

output = (1.0 − bodyResonance) × ksOutput + bodyResonance × bandpass(ksOutput, bodyFreq)
  • bodyResonance = 0.0 (default): Off — no body coloring
  • bodyResonance = 0.3–0.5: Subtle body warmth
  • bodyResonance = 0.8–1.0: Pronounced resonant body

The bodyFreq sets the center frequency of the resonant peak (80–400 Hz). This range covers typical instrument body resonances:

bodyFreq Instrument character
80–120 Hz Deep bass body (bass guitar, cello body)
150–200 Hz Mid-bass warmth (acoustic guitar body)
200–280 Hz Upper-bass boxiness
280–400 Hz Bright body ring (smaller instruments)

Body resonance is independent of bodyFreq — it only controls how much of the resonated signal is mixed in.

String Damping Flag

The stringDamping parameter (0 or 1) is a special audio-engine flag, not a sound-shaping parameter. When set to 1, the audio engine recognizes this Karplus Strong container as using a string-damping behavior, which affects how the tail/ring-out is handled at note end.

This has no direct effect on the sound within a note — its purpose is to signal graph-level rendering decisions. Leave at 0 (default) unless you specifically need the string-damping tail behavior.

Legato Behavior

For legato note transitions (notes connected without a gap), the Karplus Strong string is not reset. The string’s vibrating state carries over from the previous note — the delay line, write index, peak amplitude tracker, and filter states are all preserved. Only the pitch (fractional delay length) is updated via setPitch().

This allows realistic portamento effects: the string continues vibrating, and its harmonic character evolves as it adjusts to the new pitch. For non-legato notes, the string is fully reset and re-triggered at the new note’s pitch.

Attack Duration

attackSamples = attackPortion × 1500 × delayLength

Where 1500 is an empirical constant representing the typical number of cycles for a full K-S decay. delayLength = round(sampleRate / pitch).

attackPortion Effect
0.01–0.05 Very short pluck — barely audible transient
0.15 (default) Natural pluck onset
0.3–0.5 Extended pluck with long decay
0.7–1.0 Full K-S string sound

[Envelope Engine] → controlOut → blendRatio
                                 [Karplus Strong]

                                  pluckHardness: 0.5

The envelope sweeps the excitation from all-noise to all-soft-pluck across the note’s duration, creating a character that starts gritty and ends clean. Note: blendRatio is read at trigger time — connect before note start.

Pitch-Tracking Body Resonance

[Frequency Mapper] → controlOut → bodyFreq
  outputMin: 80, outputMax: 350
  curveType: Logarithmic
             [Karplus Strong]
              bodyResonance: 0.4

The body resonance frequency tracks the note pitch, keeping the resonant peak in a musically relevant relationship with the fundamental.

Vibrato via LFO

[LFO] → valueOut → pitchMultiplier → [Karplus Strong]
  LFO frequency:  5.5 Hz
  LFO amplitude:  0.015         (±1.5% — subtle vibrato)
  LFO center:     1.0           (offset so modulation swings around 1.0)

The LFO modulates the pitch ratio around 1.0, producing vibrato without any semitone assumptions. Ratio width is tuning-agnostic — 0.01 amplitude gives the same proportional interval at any pitch.

Static Transpose

[Karplus Strong]
  pitchMultiplier: 1.5   (just fifth up)
  pitchMultiplier: 0.667 (just fifth down)
  pitchMultiplier: 2.0   (octave up)
  pitchMultiplier: 0.5   (octave down)

Set via the inspector slider. Useful when layering two Karplus Strong containers at different intervals, or when the score is in a register that needs the string model to operate at a more comfortable pitch.

Tail Behavior

The Karplus Strong container produces a decay tail (recognized by hasTail()). Kala will render additional tail samples after a note ends to capture the full string ring-out, so the decay is heard naturally even for short notes.

Troubleshooting

No Sound

  1. Verify the trigger is firing (note start triggers automatically for non-legato notes)
  2. Check attackPortion is not too low
  3. In Attack mode, confirm mix > 0 and signalIn is connected

Sound Too Dark / Dull

  1. Increase brightness — the loop filter is losing high frequencies too fast
  2. Decrease pluckPosition toward 0.0 (bridge) — brighter noise excitation
  3. Decrease pickDirection toward 0.0 — pick direction filter adds brightness
  4. Try excitationType Noise or Blend with low blendRatio

Sound Too Bright / Harsh

  1. Decrease brightness
  2. Increase pluckPosition toward 1.0 (neck)
  3. Increase pickDirection toward 1.0 — warmer pick direction filter
  4. Use Soft Pluck or Finger Pluck excitation (less high-frequency content than Noise)
  5. Add body resonance at a low frequency to add warmth

Decay Too Fast

  1. Decrease damping — damping = 0.0 gives the longest possible sustain
  2. Increase attackPortion to include more of the K-S decay

Decay Too Slow

  1. Increase damping
  2. Decrease attackPortion

Body Resonance Not Audible

  1. Increase bodyResonance toward 0.4–0.8
  2. Ensure bodyFreq is in a range where the string has harmonic content — very high pitches have few harmonics in the 80–400 Hz range
  3. Try different bodyFreq values to find where the resonance activates

Clicks at Attack End (Attack Mode)

This should not occur due to the built-in cosine fade over the final 10% of the attack phase. If clicks occur, try slightly increasing attackPortion.

Technical Notes

Fractional Delay and Pitch Tracking

The delay line uses fractional delay with linear interpolation between adjacent samples:

readPos = writeIndex − currentDelay
sample  = delayLine[idx0] × (1 − frac) + delayLine[idx1] × frac

This allows currentDelay = sampleRate / pitch to be a non-integer value, enabling continuous pitch tracking. When pitch changes during a note (e.g., pitch curves in the Score Canvas), setPitch() updates currentDelay without re-triggering the pluck.

Output Normalization

Output is continuously normalized against a tracked peak amplitude:

output = currentSample / peakAmplitude × TARGET_PEAK   (TARGET_PEAK = 0.8)

The peak tracker runs across the entire attack duration, so the initial cycles (which may be loud) are correctly normalized even if the excitation is not uniform.

Body Resonance is Post-Loop

The body resonance biquad is applied after the K-S loop output, not inside the feedback loop. This means it colors the output without affecting the string’s decay behavior — body resonance does not change the pitch, damping, or ring-out of the string itself.
