Wavetable Synth Documentation

Overview

The Wavetable Synth is a wavetable oscillator that plays back cyclic waveforms with band-limited aliasing control and optional multi-frame morphing. Six built-in presets (Saw, Square, Triangle, Pulse, SuperSaw, Formant) are available immediately, and custom wavetables can be loaded from WAV files for fully user-defined timbres.

It outputs both a signal (direct audio playback) and a spectrum (FFT-derived harmonic amplitudes), so it can feed directly into signal chains or spectrum-processing chains, or both simultaneously.

Category

Essential (Blue) - Core sound generation

Ports

Input Ports

Port Data Type Default Description
position Control 0.0 Frame morph position (0.0–1.0). Interpolates between wavetable frames when the table has multiple frames
pitchMultiplier Control 1.0 Scales the note frequency before synthesis. 1.0 = no change, 2.0 = octave up, 0.5 = octave down. Any ratio works — the system is tuning-agnostic. Can be driven by LFO (vibrato), Envelope Engine (pitch glide), or Physics System (spring pitch wobble).

Note: The base pitch is provided automatically by the audio graph. The pitchMultiplier port scales it before playback, enabling vibrato, pitch glide, and static transposition.

Output Ports

Port Data Type Description
signalOut Signal Audio-rate waveform output (-1.0 to 1.0)
spectrum Spectrum FFT-derived harmonic amplitudes at the current position

Both ports are active when outputMode is set to Both (default). Either port can be disabled individually via outputMode.

Parameters

Parameter Type Default Range Description
wavetableSelect Integer 0 (Saw) 0–5 Built-in preset selection (ignored when a WAV file is loaded)
position Float 0.0 0.0–1.0 Frame morph position (overridden by position input when connected)
cycleLength Integer 2048 — Samples per single wavetable frame (set via inspector)
outputMode Integer 0 (Both) 0–2 Controls which output ports are active
pitchMultiplier Float 1.0 0.25–4.0 Pitch ratio applied before synthesis (1.0 = no change, 2.0 = octave up, 0.5 = octave down)

Built-In Presets

0: Saw

Classic sawtooth wave generated by additive synthesis:

amplitude = 1/n  for harmonics n = 1, 2, 3, ...

Character: Bright, buzzy, rich in harmonics. The most harmonically dense of the presets. Use for: Strings, brass-like tones, bright pads, leads.

1: Square

Square wave generated by additive synthesis (odd harmonics only):

amplitude = 1/n  for harmonics n = 1, 3, 5, 7, ...

Character: Hollow, woody quality with strong odd harmonics. Use for: Clarinets, reed-like timbres, nasal pads.

2: Triangle

Triangle wave (odd harmonics with alternating phase and 1/n² rolloff):

amplitude = 1/n²  for harmonics n = 1, 3, 5, ...  (alternating sign)

Character: Smooth, flute-like. Much softer than Square due to rapid harmonic rolloff. Use for: Flutes, soft leads, gentle sub-bass.

3: Pulse

Pulse wave with 25% duty cycle:

amplitude = (2 / (n × π)) × sin(n × π × 0.25)

Character: Thinner than square, nasal quality. The duty cycle produces a characteristic “hollow” sound. Use for: Harpsichord-like tones, thin electronic leads.

4: SuperSaw

Seven detuned sawtooth waves stacked together:

Detune offsets: −11%, −6%, −2%, 0%, +2%, +6%, +11%

Character: Massive, chorus-like detuned texture. The defining sound of classic trance synths. Use for: Lush pads, walls of sound, electronic leads with width.

5: Formant

Harmonics shaped by two Gaussian spectral bumps approximating the “ah” vowel:

F1 ≈ 5th harmonic  (−3 dB gaussian bump)
F2 ≈ 9th harmonic  (−3 dB gaussian bump)

Character: Vocal, nasal, with built-in formant color. The bump positions are fixed at preset load. Use for: Voice-like timbres, vowel textures, organic pads.

Output Mode

Value Name signalOut spectrum Use Case
0 Both Active Active Feed both signal and spectrum chains simultaneously
1 SignalOnly Active Inactive Pure signal output — skip spectrum computation
2 SpectrumOnly Inactive Active Feed into spectrum-processing chain (Rolloff, Spectrum Blender, etc.)

Note: Even in SpectrumOnly mode, the phase accumulator still advances each sample to keep playback in sync.

Position and Frame Morphing

The position parameter selects where within the wavetable’s frame stack playback occurs. Preset wavetables have a single frame (position is irrelevant unless a multi-frame WAV is loaded). Multi-frame WAV files unlock smooth morphing:

frame_pos = position × (num_frames - 1)
output    = lerp(frame[floor(frame_pos)], frame[ceil(frame_pos)], frac)

Frame interpolation is bilinear: it interpolates both across frames and within the cycle (sample interpolation), producing smooth morphing without zipper noise.

Modulating Position

Connect any control source to the position input port to animate the morph point:

[Envelope Engine] → controlOut → position → [Wavetable Synth]

An envelope sweeping from 0.0 to 1.0 across the note duration scans through the entire wavetable, creating timbral evolution.

Loading Custom Wavetables (WAV Files)

Custom wavetables can be loaded via the inspector. The WAV file is interpreted as a sequence of single-cycle frames:

num_frames = total_samples / cycleLength
frame[n]   = samples[n × cycleLength ... (n+1) × cycleLength - 1]
  • Mono files: Loaded directly
  • Stereo files: Channels are averaged to mono
  • Normalization: Each frame is independently normalized to peak ±1.0

Set cycleLength in the inspector to match the WAV file’s cycle length before loading. Common values: 256, 512, 1024, 2048, 4096.

When a WAV file is loaded, wavetableSelect is ignored — the custom table takes priority. The built-in preset is restored if the WAV is cleared.

Anti-Aliasing (Mipmaps)

The Wavetable Synth uses mipmap anti-aliasing to prevent aliasing at high pitches. At load time, a chain of band-limited copies of the wavetable is computed:

Level 0: Full bandwidth (all harmonics up to tableSize/2)
Level 1: Harmonics up to tableSize/4
Level 2: Harmonics up to tableSize/8
...

At runtime, the mipmap level is selected based on pitch and sample rate:

safeHarmonics = sampleRate / (2 × pitch)

The lowest mipmap level whose harmonic ceiling is ≤ safeHarmonics is used. This ensures the playback never produces harmonics above Nyquist, eliminating aliasing without a runtime lowpass filter.

Spectrum Output

The spectrum output reflects the FFT-derived harmonic content of the current frame position. It is updated whenever the position value changes significantly (>1e-6). The spectrum contains up to 64 harmonics, normalized so the loudest harmonic has amplitude 1.0.

This makes the Wavetable Synth useful as a replacement for the Harmonic Generator in spectrum-processing chains:

[Wavetable Synth] → spectrum → [Rolloff Processor] → [Spectrum To Signal] → signalOut

The Rolloff Processor and Spectrum Blender will shape the wavetable’s spectral content just as they would a Harmonic Generator’s output.

Usage Patterns

Simple Wavetable Oscillator

[Wavetable Synth] → signalOut → [Output]
  wavetableSelect: 0 (Saw)
  outputMode: 1 (SignalOnly)

Wavetable into Spectrum Chain

[Wavetable Synth] → spectrum → [Rolloff Processor] → [Spectrum To Signal] → signalOut
  wavetableSelect: 2 (Triangle)
  outputMode: 2 (SpectrumOnly)

Morphing Pad with Envelope

[Envelope Engine] → controlOut → position → [Wavetable Synth]
                                                   ↓ signalOut
                                             [Formant Body] → signalOut

The envelope sweeps through the wavetable frames over the note duration, creating a sound that evolves timbrally from start to finish.

SuperSaw Pad

[Wavetable Synth] → signalOut → [Rolloff Processor (signalIn)] → signalOut → [Output]
  wavetableSelect: 4 (SuperSaw)
  outputMode: 1 (SignalOnly)

The signal-mode Rolloff Processor shapes the high end of the supersaw.

Vibrato via Pitch Multiplier

[LFO] → valueOut → pitchMultiplier → [Wavetable Synth]
  LFO frequency:  5.5 Hz
  LFO amplitude:  0.015
  LFO center:     1.0

Drives frequency vibrato via the pitch ratio port. Works with any wavetable preset and is independent of the built-in envelope or morphing.

Parallel Wavetable + Harmonic Generator

[Harmonic Generator] → [Spectrum To Signal] → signalA → [Signal Mixer] → [Output]
                                                                 ↑
[Wavetable Synth (SuperSaw)] → signalOut → signalB ─────────────┘

Blend the wavetable texture with the harmonic generator’s clean tone.

Comparison with Other Generators

Container Domain Pitch Source Spectral Control
Wavetable Synth Signal + Spectrum Auto (note pitch) Preset/WAV waveform; position morphing
Harmonic Generator Spectrum Auto Per-harmonic amplitude sliders
Karplus Strong Signal Auto Pluck position, damping
Noise Color Filter Signal — Noise color/filter type

The Wavetable Synth is the fastest way to get classic electronic waveforms. For harmonic control more granular than a preset shape, use the Harmonic Generator instead.

Technical Notes

Phase Accumulation

The phase accumulator runs in the range [0, cycleLength):

phaseIncrement = pitch × cycleLength / sampleRate
phase += phaseIncrement
while (phase >= cycleLength) phase -= cycleLength

Sample interpolation within the table is linear (fractional phase).

Reset Behavior

reset() returns phase to 0.0. Called at note start (but not for legato transitions). This means all notes start at the same phase position in the waveform.

Spectrum Cache

The spectrum is lazily recomputed only when position changes by more than 1e-6. This avoids FFT overhead for static-position playback.

CPU Cost

  • Single-frame presets: Very lightweight — one table lookup with linear interpolation per sample
  • Multi-frame WAV: Same cost as single-frame (bilinear interpolation adds one multiply per sample)
  • Mipmap selection: Computed once per container execution (not per sample)
  • Spectrum output: FFT runs only when position changes — negligible for slow or static position
