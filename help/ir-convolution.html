IR Convolution Documentation

Overview

The IR Convolution container applies an impulse response (IR) to any input signal using partitioned FFT convolution. This is the standard technique for realistic reverb, room simulation, and cabinet/body modeling — the impulse response captures the acoustic “fingerprint” of a real space or physical resonator, and convolution applies that fingerprint to the dry signal.

The IR can come from two sources: a loaded WAV file (classic reverb/cabinet use), or a live signal from any other container via the irIn port (cross-synthesis). When irIn is connected, the graph is first pre-rendered for captureLength seconds at the note's pitch to capture that signal as the IR, which is then used for the full note — so the timbral and resonant character of one instrument is applied to another.

Four post-processing parameters shape the wet signal: high-frequency damping, low-cut filtering, pre-delay, and exponential decay trim.

Wet/Dry Blend

signalOut = signalIn × (1.0 - wetDry) + wet_signal × wetDry
wetDry Effect
0.0 Fully dry — input signal passes through unchanged (no convolution applied)
0.25 Mostly dry with subtle space
0.5 (default) Equal blend
0.75 Mostly wet — strong reverb/effect character
1.0 Fully wet — only the convolved signal

Pre-Delay

Pre-delay adds a gap before the wet (reverbed) signal begins. The dry signal passes through immediately; the wet signal is delayed by predelay milliseconds.

Time:    0ms         predelay         predelay + IR length
         │               │                │
Dry:     ████████████████████████████████
Wet:                     ████████████████(fade-in)█████████████
predelay (ms) Effect
0 (default) Wet signal starts immediately with the dry signal
5–20 Subtle separation — natural room feel
20–60 Clearly separated reverb onset — useful for drum rooms
60–200 Slapback + reverb effect
200–500 Long pre-delay for special effects

Fade-In Ramp (fadeInPct)

When predelay > 0, the wet signal can fade in linearly over the first fadeInPct percent of the pre-delay window. This prevents the abrupt onset of the reverb tail from sounding unnatural (an artifact of sample-accurate convolution starting mid-buffer).

  • 0%: No fade — wet signal appears at full level immediately after pre-delay
  • 10% (default): Short ramp — smooth onset of reverb
  • 50%: Long ramp — gradual swell of reverb
  • 100%: Full pre-delay used as ramp — very gentle bloom

High-Frequency Damping (highDamp)

A one-pole lowpass filter applied to the wet signal only. Frequencies above highDamp are progressively attenuated.

highDamp (Hz) Effect
20000 (default) No damping — full bandwidth wet signal
8000 Gentle darkening — models soft wall surfaces
4000 Moderate damping — warm, padded room character
2000 Strong damping — very warm, muffled reverb
500–1000 Extreme damping — submerged or heavily padded effect

High damping is useful when the IR contains too much brightness for the application, or to model acoustically absorbent materials (carpets, heavy drapes, foam).

Low-Cut Filter (lowCut)

A one-pole highpass filter applied to the wet signal only. Frequencies below lowCut are progressively attenuated.

lowCut (Hz) Effect
20 (default) No low-cut — full bandwidth wet signal
80–150 Removes sub-bass from reverb — cleaner low end
200–400 Removes bass warmth from reverb — tight, precise reverb
500–800 Strong low-cut — only upper harmonics reverberate

Low-cut prevents the reverb tail from building up muddy low-frequency content, especially important when multiple voices are playing simultaneously.

Decay Trim (decay)

An exponential fade applied to the IR itself before the partitioned FFT convolution is computed. This effectively shortens the reverb tail without changing its early-reflection character.

IR[n] = IR[n] × exp(−decay_rate × n)
decay Effect
1.0 (default) Full IR — maximum reverb length and density
0.8 Slightly shortened tail
0.5 Half the effective reverb time
0.2 Very short — only early reflections remain
0.0 Maximum shortening — nearly impulse-like

Decay changes trigger IR re-preparation (the partitioned FFT chain is recomputed), so changes during playback may cause a brief interruption. Set decay before playback.

Loading Impulse Responses

IR files are loaded via the inspector. Accepted format: WAV (any bit depth, any sample rate; stereo files are averaged to mono).

The IR file is interpreted as a linear sequence of samples describing how the space or device responds to an impulse. No special formatting is needed — any standard IR WAV file from third-party libraries will work.

Typical IR Sources

  • Room and hall IRs (reverb)
  • Speaker cabinet IRs (guitar/bass amp simulation)
  • Body resonance IRs (acoustic instrument body)
  • Hardware unit IRs (convolution synths, vintage plate reverbs)

Without an IR File

If no IR is loaded, the container passes the input signal through unchanged (signalOut = signalIn). wetDry has no effect without an IR. The inspector will indicate that no IR is loaded.

Usage Patterns

Basic Room Reverb

[Signal Source] → signalIn → [IR Convolution] → signalOut
                               wetDry: 0.3
                               predelay: 15 ms
                               highDamp: 6000 Hz
                               lowCut: 80 Hz
                               [Load a room IR]

Subtle room ambience: dry signal dominates, reverb adds spatial depth.

Large Hall

[Signal Source] → signalIn → [IR Convolution] → signalOut
                               wetDry: 0.5
                               predelay: 40 ms
                               highDamp: 12000 Hz
                               lowCut: 60 Hz
                               decay: 0.85
                               [Load a large hall IR]

Cabinet / Body Simulation

[Karplus Strong] → signalIn → [IR Convolution] → signalOut
                                wetDry: 0.85
                                predelay: 0 ms
                                highDamp: 8000 Hz
                                lowCut: 120 Hz
                                decay: 0.7
                                [Load a guitar body IR]

High wet/dry for a cabinet sim — the IR provides most of the tonal character. Short decay to avoid excessive tail.

Wet/Dry Modulation with Envelope

Reverb amount that increases toward the end of a note:

[Envelope Engine] → controlOut → wetDry → [IR Convolution]
  (envelope rises from 0.0 to 0.6 over note duration)

The note starts dry and blooms into the reverb space as it progresses.

Pitch-Adaptive Reverb Mix

More reverb on high notes:

[Frequency Mapper] → controlOut → wetDry
  outputMin: 0.1, outputMax: 0.7
  curveType: Logarithmic

High notes are more reverberated; low notes remain relatively dry.

Pre-Delay for Separation

wetDry: 0.4
predelay: 30 ms
fadeInPct: 20

30 ms pre-delay with a 20% fade-in ramp (6 ms ramp) creates a clearly separated reverb that doesn’t blur the attack of short sounds.

Cross-Synthesis with Another Instrument

[Recorder]  → irIn     → [IR Convolution] → signalOut
[KS Attack] → signalIn →
                          captureLength: 2.0
                          wetDry: 1.0

The spectral and resonant character of the Recorder is imprinted onto the Karplus Strong attack. The convolution operates at the note's pitch, so resonant modes in the captured IR naturally reinforce harmonics of the main signal.

Live Signal Cross-Synthesis

When the irIn port is connected, IR Convolution enters cross-synthesis mode. The WAV file IR (if any) is replaced for that note by a live capture of the irIn signal.

How It Works

Because Kala pre-renders all notes, the capture can be done cleanly before any audio is produced:

  1. Pre-roll: Before rendering each note, a temporary clone of the graph is run silently for captureLength seconds at the note's pitch.
  2. Capture: The irIn source's output during that pre-roll is recorded into a ring buffer.
  3. Commit: The captured signal is set as the IR, with full partitioned FFT preparation.
  4. Render: The note renders from sample 0 using the captured IR — no mid-note switch, no artifact.

Each note independently captures its own IR at its own pitch, so the cross-synthesis is pitch-coherent: a Recorder or reed model captured at A4 produces a different IR than one captured at C3.

captureLength as a Timbral Control

captureLength What is captured Character
0.1–0.5 s Attack transient Sharp, percussive coloring
1–3 s Early sustain Tonal body of the source
5–20 s Full sustained tone or decay Deep resonant fingerprint; long pre-render time

irIn vs. File IR

The two sources are mutually exclusive per note: if irIn is connected, the file IR is ignored for that note. To use a file IR, leave irIn unconnected.

Render Time

The pre-roll runs the full signal graph for captureLength × sampleRate extra samples per note. A 20-second capture at 44100 Hz adds ~880,000 samples of pre-render per note. For large scores with long capture lengths, overnight rendering may be appropriate.

Tail Behavior

IR Convolution is recognized by the audio graph as a tail-producing container (via hasTail()). After a note ends, Kala renders additional tail samples to capture the full reverb decay — so the reverb tail is heard completely even if the note is short.

The tail length is determined by the IR length and the decay setting.

Comparison with Other Spatial / Coloring Containers

Container Type Character
IR Convolution True convolution reverb Realistic space, cabinet, room
Comb Filter (Feedback) Delay-based resonator Metallic, pitched reverb
Formant Body Resonant filter Instrument body resonance
Breath Turbulence Signal mixer Voice/air texture blending

IR Convolution is the most realistic but also the most CPU-intensive option. For abstract metallic reverb effects, a Comb Filter is much lighter. For instrument body resonance, Formant Body may be more controllable.

Troubleshooting

No Reverb Effect

  1. Confirm an IR file is loaded (the inspector shows the IR file path or “No IR loaded”), or that irIn is connected
  2. Check that wetDry is > 0.0
  3. Verify signalIn is connected and producing audio

irIn Connected but No Cross-Synthesis Effect

  1. Confirm the irIn source is a signal-producing container (outputs to signalOut). Control-only sources like LFO will produce a silent IR.
  2. Check that captureLength is shorter than the note duration — if the note ends before the capture threshold is reached, the pre-roll completes correctly, but very short notes combined with long capture lengths may produce thin IRs.
  3. Set wetDry to 1.0 to hear only the convolved signal while testing.

Muddy Low End

  1. Increase lowCut to 100–200 Hz to remove sub-bass from the reverb tail
  2. Reduce wetDry slightly

Too Bright / Harsh Reverb

  1. Reduce highDamp — try 4000–8000 Hz for darker character
  2. Some IRs are inherently bright; this parameter compensates

Reverb Too Long

  1. Reduce decay to shorten the IR tail
  2. Note: decay changes require IR re-preparation — set before playback

Click at Reverb Onset

  1. Increase fadeInPct to extend the fade-in ramp after pre-delay
  2. Ensure predelay is > 0 when using fadeInPct (fade-in only applies during the pre-delay window)

High CPU Usage

  1. IR Convolution is the most CPU-intensive container. Long IRs with many partitions are expensive.
  2. Use decay to shorten the effective IR length
  3. Avoid using multiple IR Convolution containers simultaneously
  4. High-frequency damping and low-cut add minimal overhead (simple one-pole filters)

Technical Notes

Algorithm: Partitioned Overlap-Save FFT Convolution

The container uses a partitioned overlap-save algorithm for efficient convolution of long IRs:

  1. The IR is divided into blocks of blockSize samples (default: 512)
  2. Each block is FFT’d into frequency-domain IR partitions
  3. During playback, input samples are accumulated in blocks
  4. Each block: FFT the input, multiply with all IR partition spectra, accumulate, IFFT, overlap-add
  5. One output sample is retrieved from the output buffer each call

This approach has O(N log N) complexity per output sample rather than O(N) for direct convolution, making long IRs practical. The FFT is a simple radix-2 Cooley-Tukey implementation with no external dependency.

Pre-Allocated Buffers

All FFT work buffers (m_fftRe, m_fftIm, m_accumRe, m_accumIm) are pre-allocated at IR load time to avoid per-block memory allocation during audio rendering.

Reset Behavior

All internal buffers (pre-delay circular buffer, FFT delay line, output buffer, overlap buffer, damping and low-cut filter states) are zeroed at note start for non-legato notes. Preserved for legato transitions.
