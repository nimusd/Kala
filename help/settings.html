Sound Engine Settings Tab Documentation

Overview

The Settings tab provides control over project-level audio configuration and container parameter ranges. Each container setting defines the slider range exposed in the inspector — adjusting these lets you expand or constrain how far each parameter can be pushed without changing the parameter itself.

Location

  • Tab Index: 2 (Preferences)
  • Parent Component: KalaMain
  • Settings Backend: ContainerSettings singleton with QSettings persistence

Architecture

Settings Management System

  • Singleton Pattern: Global settings accessible via ContainerSettings::instance()
  • Modular Design: Separate parameter structures for each container type
  • Persistent Storage: Uses QSettings with group-based organization
  • JSON Export/Import: Supports project file serialization via toJson() / fromJson()
  • Signal-Based Updates: Emits settingsChanged() on modifications

Tab Features

  • Project Settings: Composition name, sample rate, bit depth
  • Container Parameter Ranges: Min/max limits for every container type's inspector sliders
  • Save Settings: Export all ranges to a .csettings file
  • Load Settings: Import ranges from a .csettings file
  • Reset to Defaults: Restore all ranges to their factory values

Implementation Details

Parameter Updating Flow

  1. User adjusts spinbox in Settings tab
  2. Lambda callback → ContainerSettings field updated
  3. saveSettings() called — persisted to QSettings immediately
  4. Inspector panels use the stored limits as slider bounds when created

Data Persistence

  • Storage: QSettings (system registry / ini file depending on platform)
  • Export: Save Settings button → .csettings JSON file
  • Import: Load Settings button → reads a .csettings file
  • Reset: Reset to Defaults button → restores all factory values and repopulates the tab

Integration Points

  • Inspector panels: Use ContainerSettings min/max as slider bounds when building the inspector for a container
  • KalaMain: Calls populateSettingsTab() to build the Settings UI from ContainerSettings structures
  • Project Files: Settings serialized alongside project data via toJson() / fromJson()
