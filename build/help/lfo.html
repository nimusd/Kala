LFO Documentation

Overview

The LFO (Low Frequency Oscillator) is a stateful periodic oscillator that generates cycling waveforms for modulation. Unlike the stateless Easing Applicator and Envelope Engine, the LFO maintains internal phase state and advances automatically each sample, producing continuous oscillation independent of note progress.

The output oscillates between -amplitude and +amplitude (bipolar), making it suitable for modulation that needs to swing above and below a center value.

Category

Modifiers (Green) - Generates periodic control signals for modulation

Core Concept

valueOut = generateWaveform(phase) × amplitude
phase += frequency / sampleRate  (each sample)

The LFO continuously cycles through its waveform at the specified frequency, outputting values scaled by amplitude.

Ports

Input Ports

Port Data Type Default Description
frequency Control 1.0 Oscillation frequency in Hz (0.01-100.0)
amplitude Control 1.0 Output amplitude (0.0-1.0)

Note: There is no phase input port. Phase is managed internally and advances automatically.

Output Ports

Port Data Type Description
valueOut Control Oscillating value from -amplitude to +amplitude

Parameters

Parameter Type Default Range Description

Waveform Types

0: Sine

Smooth, continuous oscillation.

     ╭─╮
    ╱   ╲
───╱─────╲───  0
            ╲   ╱
             ╰─╯
  • Formula: sin(2π × phase)
  • Output: Smoothly varies between -1 and +1
  • Character: Natural, organic movement
  • Use cases: Vibrato, gentle tremolo, filter sweeps

1: Triangle

Linear ramps between extremes.

    /\
   /  \
──/────\──  0
        \  /
         \/
  • Formula: Piecewise linear (0→1→-1→0)
  • Output: Sharp peaks, linear transitions
  • Character: Mechanical, predictable
  • Use cases: Rhythmic effects, linear sweeps

2: Square

Instant transitions between extremes.

   ┌───┐
   │   │
───┘   └───  0
       ┌───┐
       │   │
  • Formula: +1 for first half, -1 for second half
  • Output: Only two values (+amplitude and -amplitude)
  • Character: Abrupt, digital
  • Use cases: Trills, gating, rhythmic on/off effects

3: Sawtooth

Linear rise with sudden reset.

      /│
     / │
────/──│──  0
   /   │
  /    │
  • Formula: 2 × phase - 1 (rises from -1 to +1)
  • Output: Continuous rise, instant drop
  • Character: Asymmetric, building tension
  • Use cases: Risers, one-directional sweeps

Output Range

Important: The LFO output is bipolar (centered at zero):

Amplitude Output Range
1.0 -1.0 to +1.0
0.5 -0.5 to +0.5
0.25 -0.25 to +0.25

When connecting to parameters that expect 0-1 range, you may need to: 1. Use a connection function that offsets the value 2. Connect to a parameter that accepts bipolar modulation 3. Scale and offset downstream

Usage Patterns

Basic Vibrato

Modulate pitch with a slow sine wave.

LFO ──→ valueOut ──→ Spectrum to Signal (pitch, via modulation)
  │
  └── frequency: 5.0, amplitude: 0.02, waveType: 0 (Sine)

The pitch will oscillate ±2% around the base frequency at 5 Hz.

Tremolo

Modulate amplitude with a sine wave.

LFO ──→ valueOut ──→ [Amplitude/gain parameter]
  │
  └── frequency: 4.0, amplitude: 0.3, waveType: 0 (Sine)

Filter Wobble

Modulate filter cutoff with a triangle wave.

LFO ──→ valueOut ──→ Rolloff Processor (lowRolloff)
  │
  └── frequency: 0.5, amplitude: 0.5, waveType: 1 (Triangle)

Rhythmic Gating

Use square wave for on/off effects.

LFO ──→ valueOut ──→ [Gate/mix parameter]
  │
  └── frequency: 2.0, amplitude: 1.0, waveType: 2 (Square)

Riser Effect

Use sawtooth for continuously building tension.

LFO ──→ valueOut ──→ [Intensity parameter]
  │
  └── frequency: 0.25, amplitude: 1.0, waveType: 3 (Sawtooth)

One complete sweep every 4 seconds.

LFO Modulating LFO

Create complex modulation by having one LFO control another’s frequency.

LFO 1 (slow) ──→ valueOut ──→ LFO 2 (frequency input)
     │                              │
     └── freq: 0.1                  └──→ valueOut ──→ [Target]

The LFO is unique in: - Bipolar output (swings positive and negative) - Self-running (doesn’t need external progress input) - Multiple waveforms (Sine, Triangle, Square, Sawtooth)

Important Behaviors

Phase Continuity

The LFO’s phase continues advancing across notes. It does not reset at note boundaries unless explicitly reset. This means: - Long notes will see multiple LFO cycles - Short notes may only see part of a cycle - The LFO phase at note start depends on timing

Reset Behavior

The reset() method returns phase to 0. This is called when the audio engine resets, not at each note start.

Amplitude Clamping

Amplitude is clamped to 0.0-1.0. Values outside this range are brought to the boundaries.

Frequency Clamping

Frequency is clamped to 0.01-100.0 Hz. Very low frequencies create ultra-slow sweeps; high frequencies approach audio rate.

Technical Notes

Phase Accumulation

The LFO uses normalized phase (0.0 to 1.0) rather than radians:

phase += frequency / sampleRate;
while (phase >= 1.0) phase -= 1.0;

This prevents floating-point precision issues over long playback times.

Waveform Generation

Each waveform is generated from the normalized phase: - Sine: sin(2π × phase) - single call to sin() - Triangle: Piecewise linear with 4 segments - Square: Simple threshold test - Sawtooth: 2 × phase - 1

Performance

  • Lightweight per-sample processing
  • Single trig function call (sine only)
  • No memory allocation
  • Efficient for many simultaneous instances
