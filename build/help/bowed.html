Bowed Documentation

Overview

The Bowed container is a digital waveguide physical model of a bowed string instrument, based on the algorithm developed by Perry R. Cook and Gary P. Scavone in the Synthesis ToolKit (STK) and extended with the nonlinear waveguide technique by Romain Michon. It simulates the interaction between a bow and a string: a nonlinear friction table drives two coupled delay lines representing the string segments on either side of the bow contact point, filtered through a cascade of body resonance filters derived from measured violin impulse response data.

Because the model is self-oscillating, it produces a continuous pitched tone for the full duration of any note. The pitch is tracked continuously, so pitch curves and portamento in the Score Canvas work without artifacts. Legato transitions preserve the vibrating string state, allowing smooth slurs between notes. The optional nonlinear allpass section (Michon extension) can push the timbre from clean and violin-like into progressively richer, grittier, or FM-adjacent territory.

How It Works

The algorithm is a two-segment digital waveguide simulation of bowed-string acoustics. Each sample is computed as follows:

bowAmp         = adsrEnvelope() × (0.03 + 0.2 × bowVelocity)

nutReflection  = −neckDelay.lastOut
bridgeRefl     = −stringFilter( bridgeDelay.lastOut )
stringVel      = bridgeRefl + nutReflection

deltaV         = bowAmp − stringVel
x              = deltaV × slope + 0.001   (slope = 5 − 4×bowPressure)
friction       = clamp( (|x| + 0.75)⁻⁴,  0.01, 0.98 )
newVelocity    = deltaV × friction

neckDelay.tick( bridgeRefl + newVelocity )
bridgeDelay.tick( nutRefl  + newVelocity )

nlSig          = nonlinearAllpass( bridgeDelay.lastOut,  θ )
output         = bodyFilters( nlSig ) × 0.1248

The key components are:

  • Neck delay: Models the string segment from the bow contact point to the nut (or fingered position). Its length is (1 − bowPosition) × baseDelay samples.
  • Bridge delay: Models the shorter segment from the bow to the bridge. Length is bowPosition × baseDelay samples. baseDelay = sampleRate / pitch − 4, where the −4 compensates for filter latency.
  • Bow table (friction model): Implements Smith's stick-slip friction model. The function (|x| + 0.75)⁻⁴ produces a characteristic curve where near-zero relative velocity yields high friction (stick phase) and large relative velocity yields low friction (slip phase). The slope parameter, controlled by bowPressure, sets the steepness of this curve.
  • String loss filter: A one-pole low-pass applied to the bridge reflection. It models the energy absorbed each time the wave reflects off the bridge end. The pole position is derived from the STK formula 0.75 − 0.2 × 22050 / sampleRate. The feedforward coefficient is gain × (1 − |pole|), ensuring DC loop gain = 0.95 < 1 so the string always decays when the bow stops.
  • Body resonance filters: Six cascaded BiQuad sections with coefficients from a measured violin body impulse response (Esteban Maestre, 2011). They shape the spectral output to give the bowed tone its characteristic resonant character without affecting the waveguide pitch.
  • Nonlinear allpass (Michon extension): A first-order allpass filter inserted between the bridge delay output and the body filters. Its coefficient θ is modulated by one of five sources. See Nonlinear Section below.

Bow Envelope

A simple ADSR envelope is applied to the bow excitation amplitude at the start and end of each note:

  • Attack (20 ms): Ramps from 0 to peak bow force, simulating the bow being placed on the string. Skipped for legato notes.
  • Decay (5 ms): Short decay to the sustain level.
  • Sustain: Bow held at 90% of peak force for the duration of the note.
  • Release (10 ms): Bow lifts off the string as the note ends.

This envelope is internal and affects only the excitation. Overall note dynamics are controlled by the Score Canvas as usual.

Nonlinear Section (Michon Extension)

When nlAmount > 0, a first-order nonlinear allpass filter is applied to the bridge delay output on each sample:

H(z) = (θ + z⁻¹) / (1 + θ·z⁻¹)
y[n] = θ·(x[n] − y[n−1]) + x[n−1]

The allpass coefficient θ is derived from one of five modulation sources, scaled by nlAmount and the ramp-in envelope, then clamped to (−0.999, 0.999) for stability:

Type Source Character
0 Bridge signal Direct signal-dependent modulation. Thickens tone and adds subtle harmonic richness that tracks the string vibration.
1 Leaky average of bridge signal Slow, DC-like drift of the allpass coefficient. Produces a gentle tonal shift over time rather than per-sample modulation.
2 Bridge signal² Always positive; responds to energy bursts in the string. Asymmetric modulation that emphasises the attack and forte moments.
3 Sine wave at nlFreqMod Hz Periodic, independent tremolo-like modulation. At low nlFreqMod values produces slow tonal wobble; at higher values produces audible sidebands.
4 Sine wave at pitch Hz Sine locked to the fundamental frequency. Produces harmonically coherent waveshaping that tracks pitch — useful for consistent timbre across the register.

The nlAttack parameter ramps the effect from zero to full depth over the specified number of seconds at the start of each note. This prevents a hard click when the nonlinearity switches on and allows the string tone to establish itself before the distortion is applied.

Tip: Start with nlType = 0 or nlType = 2 at low nlAmount (0.05–0.15) for the most musical, natural-sounding extension. Types 3 and 4 at higher amounts produce more overtly electronic or FM-like timbres.

Parameter Guide

Bow Pressure

Controls the steepness of the bow-table friction curve. This is the most character-defining parameter — it sets how strongly the bow bites into the string.

  • 0.2–0.4: Light, feathery bowing — thin and transparent tone, may not sustain at low velocities
  • 0.6–0.8: Full, centred string tone with clear fundamental (default 0.75)
  • 0.85–1.0: Heavy bowing — richer in upper partials, slightly more aggressive edge to the sound

Bow Velocity

Sets the maximum bow speed amplitude. Together with bowPressure, this is the main volume control for the Bowed container.

  • 0.1–0.3: Pianissimo — light and delicate
  • 0.4–0.6: Normal playing level (default 0.5)
  • 0.7–1.0: Forte to fortissimo

Modulating bowVelocity with a Drift Engine or LFO produces natural bow speed variation. Connecting a Physics System to bowVelocity can simulate the inertia of a real bow arm.

Bow Position

Sets where along the string the bow contacts, expressed as a ratio betaRatio: the bridge delay length = bowPosition × totalDelay.

  • 0.05–0.12: Sul ponticello — near the bridge. Bright, glassy, rich in upper harmonics; traditional for special effect
  • 0.12–0.20: Normale — standard bowing position. Warm and balanced (default 0.127)
  • 0.25–0.5: Sul tasto — over the fingerboard. Dark, hollow, flute-like tone
  • Above 0.5: Increasingly tonal instability; the bridge segment becomes dominant

This is an excellent target for slow modulation via a Physics System or Drift Engine to simulate natural bow placement variation.

NL Amount

The primary nonlinearity control. At 0.0 the nonlinear allpass is completely bypassed (zero overhead). As the value increases, the allpass coefficient modulation grows and the tone becomes progressively more complex.

  • 0.0: Pure STK bowed string — no nonlinearity
  • 0.02–0.10: Subtle warmth and harmonic enrichment, barely perceptible as a separate effect
  • 0.15–0.35: Clearly audible tonal thickening; adds life and movement to sustained notes
  • 0.5–0.8: Strong modulation — noticeable waveshaping and spectral spread
  • 0.9–1.0: Aggressive; the allpass dominates the tone; experimental territory

NL Freq Mod

Only active with nlType = 3 (independent sine) or nlType = 4 (sine at pitch Hz — the parameter is ignored for type 4 since the pitch is used instead).

  • 0.1–5 Hz: Very slow sweep — almost static tonal drift
  • 5–15 Hz: Tremolo-rate modulation; audible as a rhythmic tonal pulse
  • 30–80 Hz: Audio-rate sideband generation; additive timbre colouring
  • 80–200 Hz: Rich FM-like sidebands; highly inharmonic at high nlAmount

NL Attack

Ramps the nonlinearity from 0 to full depth at note start. A short attack (0.01–0.05 s) lets the effect appear almost immediately. A long attack (0.5–2.0 s) gradually introduces the nonlinearity during the note, which can be very expressive — the tone starts clean and grows progressively richer.

Legato Behavior

For legato note transitions the neck and bridge delay buffers are preserved. The string continues vibrating uninterrupted from the previous note; only the delay line lengths are updated to the new pitch. The bow ADSR attack phase is skipped so there is no re-onset transient between slurred notes. The nonlinear allpass state (filter memory and oscillator phase) is also preserved.

For non-legato notes, all delay lines, the string filter, the body filters, and the nonlinear allpass state are fully cleared.

Tail Behavior

When a note ends, the bow ADSR enters its release phase and the bow excitation ramps off. The waveguide continues to circulate energy, which decays each round trip due to the string loss filter (loop gain = 0.95). At typical pitches, the string rings down to silence in roughly 0.5–2 seconds, after which the audio engine detects silence and closes the tail cleanly. The nonlinear allpass remains active during the tail so the character of the decay is consistent with the sustained tone.

Very slow independent sine modulation combined with a long NL attack — the timbre evolves steadily from clean to faintly modulated over 1.5 seconds. Effective for long sustained notes in slow music.

Combining with Formant Body

[Bowed] → signalOut → [Formant Body] → signalOut → [Output]

The Formant Body processor adds vowel-like resonances that strongly reinforce the bowed tone's character. Use low f1/ f2 frequencies (200–600 Hz) for cello-like warmth, or higher values (800–2000 Hz) for viola character.

Modulating Bow Position

[LFO] → valueOut → bowPosition : [Bowed]
  LFO frequency:  0.12 Hz
  LFO amplitude:  0.06

A slow LFO on bowPosition recreates the natural wandering of a real bow along the string, producing subtle tonal variation over time.

Vibrato via Pitch Multiplier

[LFO] → valueOut → pitchMultiplier → [Bowed]
  LFO frequency:  5.5 Hz
  LFO amplitude:  0.015
  LFO center:     1.0

Drives vibrato externally via the pitch ratio port. Unlike the internal vibrato on the Recorder/Reed models, this approach works for all container types and can be shared across multiple instruments from a single LFO.

Troubleshooting

No Sound or Very Faint

  1. Increase bowVelocity — values below 0.2 can produce very weak excitation
  2. Ensure bowPressure is above 0.4; too-light pressure may not sustain stick-slip oscillation
  3. Check bowPosition is not at 0 or 1 (degenerate boundary conditions)

Tone Unstable or Noisy

  1. Reduce nlAmount — high values at unstable parameter combinations can excite the body filter resonances excessively
  2. Reduce bowVelocity slightly; very high values can cause the friction model to enter an unstable operating region
  3. Move bowPosition closer to the default (0.127); extreme positions reduce the bridge delay length and can cause aliasing at high pitches

Tone Too Bright / Harsh

  1. Increase bowPosition toward 0.20–0.30 (move away from bridge)
  2. Reduce bowPressure slightly
  3. Add a Rolloff Processor or LP/HP Filter after the Bowed container to attenuate high frequencies

Tone Too Dark / Lacks Presence

  1. Reduce bowPosition toward 0.07–0.10
  2. Increase bowPressure slightly toward 0.80–0.85

NL Effect Too Abrupt at Note Start

  1. Increase nlAttack — a value of 0.3–0.8 s gives the nonlinearity time to fade in smoothly
  2. Reduce nlAmount — even small amounts can be surprisingly audible in the attack phase

Technical Notes

Pitch Tracking

The total string delay is recalculated whenever the pitch changes:

baseDelay = sampleRate / pitch − 4.0

The −4.0 compensates for the combined latency of the string filter and body filter cascade, keeping the oscillation accurately in tune. The neck and bridge delay lengths are updated simultaneously according to bowPosition.

String Filter Stability

The one-pole string loss filter uses the STK OnePole formula:

b0  = gain × (1 − |pole|)    (= 0.95 × (1 − |pole|))
y[n] = b0 × x[n] + pole × y[n−1]

The b0 coefficient is precomputed at construction. This ensures that the DC gain of the filter equals gain = 0.95, so the waveguide loop gain is strictly less than 1 at all frequencies. Without this correction (using gain × x[n] directly) the loop gain would be > 2 and the string would diverge.

No STK Dependency

The model is implemented entirely within Kala (bowedmodel.h / bowedmodel.cpp) with no dependency on the STK library. All DSP components — the delay lines, one-pole filter, BiQuad body filters, bow table, ADSR, and nonlinear allpass — are self-contained.
