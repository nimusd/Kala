Breath Turbulence Documentation

Overview

Breath Turbulence creates breath and noise character by blending voice input with noise signal. Simulates the air turbulence and breath noise characteristics of wind instruments and human vocal production.

Category

Shaping (Orange) - Transforms sound character

Ports

Input Ports

Port Name Data Type Default Description
voiceIn Signal - Primary audio signal (typically from Formant Body or Spectrum to Signal)
noiseIn Signal - Noise signal source (from Noise Color Filter or other noise generator)
blend Control 0.1 Mix ratio (0.0 = voice only, 1.0 = noise only)

Output Ports

Port Name Data Type Description
signalOut Signal Voice signal blended with noise/turbulence character

Parameters

Parameter Type Default Range Description
blend Float 0.1 0.0 - 1.0 Voice to noise blend ratio

Processing Algorithm

Signal Blending

output = voiceIn * (1.0 - blend) + noiseIn * blend;

Simple Linear Crossfade

  • Blend = 0.0: Pure voice signal (no noise)
  • Blend = 0.5: Equal mix voice and noise
  • Blend = 1.0: Pure noise (no voice signal)

Configuration (Preferences)

No specific configuration preferences - uses standard control ranges (0.0 - 1.0).

Practical Applications

Sound Design Techniques

1. Wind Instrument Character

  • Flute: Low blend (0.05-0.15) for subtle breath
  • Saxophone: Medium blend (0.2-0.35) for more pronounced breath
  • Clarinet: Variable blend (0.1-0.4) for expressive control

2. Vocal Character

  • Whisper Effect: High blend (0.6-0.8) with noise input
  • Breathy Vocals: Low-medium blend (0.1-0.3)
  • Harsh Vocals: High blend (0.4-0.6) for aggressive character

3. Textural Effects

  • Air Noise: Pure noise with subtle voice blend
  • Wind Effects: Voice signal modulated by wind-like noise
  • Atmospheric Textures: Slowly evolving blend patterns

Performance Parameters

Blend Amount Guidelines

  • Subtle: 0.02-0.1 - Barely audible breath character
  • Natural: 0.1-0.3 - Realistic wind/vocal sounds
  • Pronounced: 0.3-0.6 - Clear breath/turbulence
  • Extreme: 0.6-1.0 - Noise-dominant effects

Noise Source Selection

  • White Noise: For harsh turbulence
  • Pink Noise: For natural wind sounds
  • Filtered Noise: For specific spectral character
  • Colored Noise: Via Noise Color Filter for precise control

Troubleshooting

No Breath Effect

  1. Verify noiseIn is connected from Noise Color Filter
  2. Check blend parameter is > 0.0
  3. Ensure voiceIn signal is present
  4. Verify output connections

Too Much Noise

  1. Reduce blend parameter toward voice-only
  2. Check noise source characteristics
  3. Consider using filtered noise instead of white noise
  4. Adjust noise color if using Noise Color Filter

Unwanted Distortion

  1. Check that signals aren’t clipping before blending
  2. Verify blend parameter isn’t too high for clean sound
  3. Ensure noise source is appropriate level
  4. Consider adding gain control before blend

Thin Sound

  1. Increase blend to add body through noise
  2. Check noise frequency content
  3. Verify voice signal has sufficient low frequencies
  4. Consider adding low-frequency boost to noise

Best Practices

Parameter Optimization

  1. Start subtle: Begin with blend 0.05-0.15
  2. Context matters: Match breath character to sound type
  3. Dynamic control: Use automation for expressive variation
  4. Noise selection: Choose appropriate noise color

Signal Chain Design

  1. Position after character: Place after Formant Body or other shaping
  2. Noise quality: Use filtered/colored noise for better integration
  3. Level matching: Balance voice and noise levels appropriately
  4. Downstream processing: May need EQ or dynamics after blend

Creative Techniques

  1. Automated breath: Use envelope for natural breath evolution
  2. Noise modulation: Apply slow LFO to blend for pulsing effects
  3. Layered effects: Use multiple instances for complex textures

Performance Considerations

  1. CPU efficiency: Very lightweight processing
  2. Signal quality: Maintain good signal-to-noise ratio
  3. Real-time safety: Avoid sudden large blend changes
  4. Multiple instances: Can use several for complex textures

Technical Notes

Signal Mathematics

Linear Interpolation

Uses simple linear crossfade between two signals:

output = voice × (1 - blend) + noise × blend

Energy Conservation

Linear blend preserves signal energy: - Blend = 0.0: Full voice signal energy - Blend = 0.5: 50% voice + 50% noise energy - Blend = 1.0: Full noise signal energy

Acoustic Principles

Wind Instrument Physics

  • Air turbulence: Creates noise component in real wind instruments
  • Breath noise: Results from air column instability
  • Player control: Real players vary breath intensity naturally

Vocal Production

  • Glottal noise: Inherent in human voice production
  • Air flow: Creates turbulence noise component
  • Expressive tool: Players/conductors use breath character intentionally

Filter Considerations

Noise Spectral Content

  • White noise: Equal energy across all frequencies
  • Pink noise: More low-frequency energy
  • Band-limited noise: Specific frequency ranges
  • Colored noise: Custom spectral shapes
