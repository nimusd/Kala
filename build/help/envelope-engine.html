Envelope Engine Documentation

Overview

The Envelope Engine is a progress-based curve evaluator that generates envelope values based on note progress (0→1). Unlike traditional gate-triggered ADSRs, the Envelope Engine doesn’t track time internally - it receives the current position in the note from the audio engine and returns the appropriate envelope value for that position.

This design integrates naturally with Kala’s note-based rendering: the audio engine knows how far through the note we are (0.0 at start, 1.0 at end), and the Envelope Engine shapes what the control value should be at each point.

Category

Modifiers (Green) - Generates time-varying control signals based on note progress

Core Concept

envelopeValue = evaluateShape(noteProgress × timeScale) × valueScale + valueOffset

The envelope shape is evaluated at the scaled progress position, then the output is scaled and offset. Different envelope types (Linear, AttackDecay, ADSR, Fade In/Out, Custom) provide different shapes.

Ports

Input Ports

Port Data Type Default Description
timeScale Control 1.0 Stretch/compress the envelope (0.1-10.0)
valueScale Control 1.0 Multiply output value (0.0-10.0)
valueOffset Control 0.0 Add to output value (-10.0 to 10.0)

Note: There is no gate or trigger input. The note progress is provided automatically by the audio engine during rendering.

Output Ports

Port Data Type Description
envelopeValue Control Evaluated envelope value for the current note position

Parameters

Main Parameters

Parameter Type Default Range Description
envelopeSelect Integer 0 0-5 Envelope type (see below)
timeScale Float 1.0 0.1-10.0 Time stretch factor
valueScale Float 1.0 0.0-10.0 Output multiplier
valueOffset Float 0.0 -10.0 to 10.0 Output offset

Envelope Shape Parameters

These parameters control the shape of certain envelope types (AttackDecay, SustainRelease):

Parameter Type Default Range Description
envAttack Float 0.1 0.001-5.0 Attack portion (normalized, 0-1)
envDecay Float 0.2 0.001-5.0 Decay portion (normalized)
envSustain Float 0.7 0.0-1.0 Sustain level
envRelease Float 0.2 0.001-5.0 Release portion (normalized)
envFadeTime Float 0.5 0.0-1.0 Fade duration for Fade In/Out types

Follow Curve

The Envelope Engine container has a small ~ toggle button in its header. This is the Follow Curve mode.

State Appearance Behavior
Off (default) Dark / transparent Normal mode — output is determined by the selected envelope shape
On Green Curve mode — the output follows one of the note's named curves instead of the envelope shape

What it does

In Kala, every note can carry any number of named curves — continuous envelopes that evolve over the note's duration (see The Curve System in the Score Canvas documentation). When Follow Curve is on, the Envelope Engine reads one of those curves sample-by-sample and re-exposes it as a control signal that you can route to any parameter in the sound graph.

A dropdown in the inspector lets you choose which named curve this instance will follow. The dropdown lists all curve names that appear on notes in the current track — for example dynamics, brightness, or any custom name you gave a curve in the score. Each Envelope Engine instance makes its own selection independently, so several instances in the same graph can each follow a different curve simultaneously.

The inspector also displays a live preview of the selected curve, so you can see its shape without switching back to the score canvas.

The envelope shape, loop mode, and timing parameters are all ignored in this mode. The output formula becomes:

output = curveValue × valueScale + valueOffset

valueScale and valueOffset still apply, so you can rescale or shift the curve's range before routing it downstream.

Why use it

This is the bridge between the expressive curves you composed on the score canvas and any synthesis parameter that can accept a control signal. Instead of a fixed envelope shape, the curves you designed per note drive whatever parameters you connect them to.

Because multiple Envelope Engine instances can each follow a different named curve, a single note can simultaneously modulate several independent parameters — dynamics driving amplitude, a brightness curve opening a filter, a body curve swelling a resonance — each with its own routing, scale, and offset.

Example uses:

  • Dynamics curve → brighter timbre (route to Harmonic Generator purity or Rolloff Processor lowRolloff)
  • A custom breathiness curve → more breath turbulence (route to Breath Turbulence amount)
  • Curve-linked filter sweeps, reverb depth, vibrato depth, etc.

Envelope Types

0: Linear

Simple linear ramp from 0 to 1.

Output: noteProgress (directly)

Use case: Basic automation, testing, linear parameter sweeps.

1: AttackDecay

Triangle-like shape: rises to peak then falls to zero.

        /\
       /  \
      /    \
     /      \
    /        \
   0──────────1

Uses envAttack and envDecay to control the peak position. The attack and decay portions are relative to each other.

Use case: Percussive sounds, plucks, impacts.

2: SustainRelease (ADSR)

Full ADSR envelope mapped to note progress.

      /‾‾‾‾‾‾\
     /        \
    /          \
   0────────────1
   A  D   S   R

Uses all timing parameters (envAttack, envDecay, envSustain, envRelease). The sustain duration is calculated as whatever remains after attack, decay, and release portions.

Use case: Sustained instruments, pads, evolving sounds.

3: ExponentialRise (Fade In)

Stays at zero, then rises to 1.

              /
             /
            /
    _______/
   0────────────1

Uses envFadeTime to control when the fade begins (as a portion from the end).

Use case: Fade-in effects, delayed entry.

4: ExponentialFall (Fade Out)

Stays at 1, then falls to zero.

   ‾‾‾‾‾‾‾\
           \
            \
             \
   0────────────1

Uses envFadeTime to control when the fade begins (as a portion from the end).

Use case: Fade-out effects, natural endings.

5: Custom

User-defined multi-point envelope with configurable curve types between points.

Each point has: - time: Position (0-1) - value: Output value - curveType: Interpolation (0=Linear, 1=Smooth/Cosine, 2=Step)

Editor interactions

Action Result
Click empty area Add a new point (Linear by default)
Drag point Move the point
Drag point off canvas Delete the point
Right-click point Delete the point
Shift+drag Snap to grid
Ctrl+click point Cycle segment curve type: Linear → Smooth → Step → Linear

Point color indicates the curve type of the segment starting at that point:

  • Blue — Linear
  • Green — Smooth (cosine ease in/out)
  • Orange — Step (hold value until next point)

Use case: Complex automation, precise control, unique shapes.

Loop Modes

The Envelope Engine supports three loop modes for when timeScale causes progress to exceed 1.0:

Mode Behavior
None Clamp to final value (default)
Loop Restart from beginning
Pingpong Reverse direction, bounce back and forth

Legato Behavior

When processing legato notes (tied notes without re-attack), the Envelope Engine can skip the attack phase and start at the post-attack level. This is handled automatically by the audio engine passing isLegato=true.

For each envelope type, the legato start position is: - Linear: 0.3 (30% through) - AttackDecay: Peak position (end of attack) - SustainRelease: After attack+decay (sustain phase) - ExponentialRise: At fadeTime - ExponentialFall: 0.0 (starts at full level anyway) - Custom: 0.1 (10% through, approximate)

Usage Patterns

Basic Amplitude Envelope

Shape note dynamics over the note lifetime.

Envelope Engine (ADSR) ──→ envelopeValue ──→ [Amplitude control]
     │
     └── envelopeSelect: 2, envAttack: 0.1, envSustain: 0.7

Filter Sweep

Animate filter cutoff from note start to end.

Envelope Engine (Linear) ──→ envelopeValue ──→ Rolloff Processor (lowRolloff)
     │
     └── valueScale: 1000, valueOffset: 200

Result: Filter cutoff sweeps from 200 Hz to 1200 Hz over the note duration.

Percussive Brightness

Fast attack, quick decay for brightness that fades.

Envelope Engine (AttackDecay) ──→ envelopeValue ──→ Harmonic Generator (purity)
     │
     └── envAttack: 0.05, envDecay: 0.3

Time-Stretched Envelope

Use timeScale to make the envelope faster or slower than the note.

Envelope Engine ──→ envelopeValue
     │
     └── timeScale: 2.0  (envelope completes in first half of note)

With timeScale: 2.0 and loop mode “None”, the envelope reaches its end at 50% note progress and holds the final value.

Looping Envelope

Create repeating patterns within a single note.

Envelope Engine (AttackDecay) ──→ envelopeValue ──→ [Tremolo/Vibrato depth]
     │
     └── timeScale: 4.0, loopMode: Loop

The attack-decay shape repeats 4 times during the note.

Curve-Driven Brightness

Route the note's dynamics curve to the harmonic generator so that louder passages automatically produce a brighter sound.

Envelope Engine (~) ──→ envelopeValue ──→ Harmonic Generator (purity)
     │
     └── Follow Curve ON, curve: "dynamics", valueScale: 1.0, valueOffset: 0.0

The ~ button in the header must be active (green) and the curve dropdown set to dynamics. The envelope shape selector is irrelevant in this mode.

Curve-Driven Filter

Scale and offset a curve's range to control a filter cutoff.

Envelope Engine (~) ──→ envelopeValue ──→ Rolloff Processor (lowRolloff)
     │
     └── Follow Curve ON, curve: "dynamics", valueScale: 800, valueOffset: 200

Result: soft passage → ~200 Hz cutoff, loud passage → ~1000 Hz cutoff. Swap the curve dropdown to a custom brightness curve to decouple the filter from amplitude entirely.

Comparison with Other Containers

Container Input Output State
Envelope Engine Note progress (implicit) Envelope value Stateless (evaluates curve)
Easing Applicator progress, startValue, endValue Interpolated value Stateless
Drift Engine amount, rate Detuning multiplier Stateful (phase)
LFO frequency, amplitude Oscillating value Stateful (phase)

The Envelope Engine and Easing Applicator are both stateless curve evaluators. The difference is: - Easing Applicator: Interpolates between two values with shaped motion - Envelope Engine: Evaluates complex envelope shapes (ADSR, custom points, etc.)

Technical Notes

Progress-Based Design

Unlike traditional ADSRs that track elapsed time and gate state, the Envelope Engine is purely functional:

double envelopeValue = process(noteProgress, isLegato);

Given the same progress value, it always returns the same envelope value. This makes it: - Deterministic: Perfect for offline rendering - Scrubable: Can evaluate any position instantly - Composable: Multiple envelopes without timing conflicts

Custom Envelope Interpolation

Custom envelopes support three interpolation modes between points:

  • Linear (0): Straight line between points
  • Smooth (1): Cosine interpolation — eases in and out, zero derivative at both endpoints. Used by the Swell and Bell Curve presets.
  • Step (2): Hold previous value until next point

The curve type is set per-point in the editor using Ctrl+click to cycle through types. The curve type on a point governs the segment from that point to the next.

Parameter Ranges

  • timeScale: Clamped to 0.1-10.0 (via setTimeScale)
  • valueScale: Clamped to 0.0-10.0 (via setValueScale)
  • valueOffset: Clamped to -10.0 to 10.0 (via setValueOffset)
  • Timing parameters (attack, decay, release): Clamped to 0.001-5.0
  • fadeTime: Clamped to 0.0-1.0

Output Range

The raw envelope output is typically 0.0-1.0, but after scaling and offset:

finalOutput = rawEnvelope × valueScale + valueOffset

This can produce any range needed for the target parameter.
