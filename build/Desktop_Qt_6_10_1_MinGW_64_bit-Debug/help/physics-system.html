Physics System Documentation

Overview

The Physics System is a stateful mass-spring-damper simulator that creates organic, physically-plausible parameter movement. It smooths abrupt changes and adds natural inertia, momentum, and settling behavior to any control signal.

This is a core “organic without random” mechanism - the output moves naturally toward targets without artificial randomness, creating smooth transitions and preventing clicks from abrupt parameter changes.

Category

Modifiers (Green) - Creates physics-based parameter smoothing and movement

Core Concept

force = (targetValue - currentValue) × springK
velocity += force / mass
velocity *= damping
currentValue += velocity

The system simulates a mass on a spring being pulled toward the target value, with damping that dissipates energy over time. This creates smooth, natural motion with overshoot, settling, and momentum.

Ports

Input Ports

Port Data Type Default Description
targetValue Control 0.0 Where the system should move toward
mass Control 0.5 Resistance to acceleration (0.0-10.0)
springK Control 0.001 Spring stiffness - pull strength (0.0001-1.0)
damping Control 0.995 Energy retention per sample (0.5-0.9999)
impulse Trigger - Trigger input for velocity kick
impulseAmount Control 100.0 Strength of impulse when triggered (0.0-1000.0)

Output Ports

Port Data Type Description
currentValue Control Current position of the simulated mass

Parameters

Parameter Type Default Range Description
mass Float 0.5 0.0-10.0 Inertia - higher = slower response
springK Float 0.001 0.0001-1.0 Spring strength - higher = faster pull
damping Float 0.995 0.5-0.9999 Energy retention - higher = longer motion
impulseAmount Float 100.0 0.0-1000.0 Velocity added on impulse trigger

Parameter Behavior

Mass

Controls resistance to acceleration (inertia).

Mass Behavior
0.0 Instant response (no inertia)
0.1-0.5 Light, responsive
0.5-2.0 Moderate inertia
2.0-10.0 Heavy, slow to accelerate

Low mass = quick response but potentially jittery. High mass = smooth but sluggish.

Spring Constant (springK)

Controls how strongly the system pulls toward the target.

springK Behavior
0.0001-0.001 Very weak pull, slow drift
0.001-0.01 Gentle pull, smooth motion
0.01-0.1 Moderate pull, responsive
0.1-1.0 Strong pull, fast tracking

Higher springK with lower damping can cause oscillation around the target.

Damping

Controls energy loss per sample. Values close to 1.0 retain more energy.

Damping Behavior
0.5-0.8 Heavy damping, quick settling, may oscillate
0.9-0.95 Moderate damping, balanced
0.95-0.99 Light damping, sustained motion
0.99-0.9999 Very light damping, long sustain

Note: Damping < 1.0 means energy is always lost. The system will eventually settle at the target.

Impulse

When the impulse input crosses 0.5 (rising edge), the impulseAmount is added to velocity. This creates sudden “kicks” that the system then naturally recovers from.

Physics Characteristics

Underdamped (Oscillation)

When springK is high relative to damping, the system overshoots and oscillates around the target before settling.

Usage Patterns

Parameter Smoothing

Eliminate clicks from abrupt parameter changes.

[Stepped input] ──→ targetValue
                          │
                          └──→ Physics System ──→ currentValue ──→ [Smooth output]
                                    │
                                    └── mass: 0.3, springK: 0.01, damping: 0.99

Organic Filter Movement

Create natural-feeling filter sweeps.

Envelope Engine ──→ targetValue ──→ Physics System ──→ currentValue ──→ Rolloff Processor
                                          │
                                          └── mass: 0.5, springK: 0.005, damping: 0.995

Pitch Glide with Momentum

Slide between pitches with physical feel.

[Note pitch] ──→ targetValue ──→ Physics System ──→ currentValue ──→ Spectrum to Signal (pitch)
                                       │
                                       └── mass: 0.2, springK: 0.02, damping: 0.98

Bouncy Parameter Response

Create springy, elastic parameter behavior.

LFO ──→ targetValue ──→ Physics System ──→ currentValue ──→ [Target]
                              │
                              └── mass: 0.5, springK: 0.1, damping: 0.9

The output will overshoot and bounce around the LFO’s value.

                                        │

Comparison with Other Modifiers

Container Behavior State
Physics System Tracks target with mass/spring physics Stateful (position + velocity)
Easing Applicator Shapes interpolation curve Stateless
Envelope Engine Evaluates envelope at progress Stateless
Drift Engine Generates wandering multiplier Stateful (phase)
LFO Generates periodic oscillation Stateful (phase)

The Physics System is unique in having two state variables (position and velocity), which allows for momentum and overshoot - something the other modifiers cannot produce.

Important Behaviors

Velocity Clamping

Velocity is clamped to ±10,000 units/sample to prevent numerical instability with extreme parameter combinations.

Mass Near Zero

When mass < 0.0001, the force is applied directly to velocity without division (preventing division by near-zero). This gives instant response.

Reset Behavior

reset() sets currentValue and velocity to 0. reset(initialValue) sets currentValue to the specified value and velocity to 0.

Impulse Detection

Impulse triggers on rising edge - when the impulse input crosses from below 0.5 to above 0.5. Holding the trigger high does not re-trigger.

Technical Notes

State Variables

The Physics System maintains two state variables: - currentValue: Current position (output) - velocity: Rate of change

Processing Per Sample

double force = (targetValue - currentValue) * springK;
if (mass > 0.0001) {
    velocity += force / mass;
} else {
    velocity += force;
}
velocity *= damping;
velocity = clamp(velocity, -10000, 10000);
currentValue += velocity;

Mathematical Model

This implements a discrete approximation of the second-order differential equation:

m × d²x/dt² + c × dx/dt + k × x = k × target

Where: - m = mass - c = damping coefficient (derived from damping parameter) - k = spring constant (springK) - x = current position - target = target position
