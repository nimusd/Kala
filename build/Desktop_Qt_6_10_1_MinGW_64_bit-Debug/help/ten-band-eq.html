10-Band EQ Documentation

Overview

The 10-Band EQ is a graphic equalizer with ten parallel bandpass filters at the ISO standard one-octave center frequencies. Each band’s gain is independently controllable and fully modulatable, making it possible to shape the spectrum of any signal dynamically — sculpting formants, boosting harmonics, or suppressing resonances with per-note expressivity.

Unlike the Rolloff Processor (which applies a spectral taper to the harmonic domain), the 10-Band EQ operates entirely in the signal domain and can be placed anywhere in a signal chain.

Category

Filters (Purple) - Signal processing and spectral shaping

Core Concept

signalOut = Σ (bandpass_n(signalIn) × gain_n)   for n = 1 to 10

The input signal is passed simultaneously through all ten biquad bandpass filters. Each filter’s output is scaled by its gain, and the ten scaled outputs are summed to form the final output.

At unity gain (all bands = 1.0), all spectral content passes through unchanged. Setting a band to 0.0 silences that frequency region; values above 1.0 boost it.

Band Q Parameters (Inspector Only)

Q values are set via the inspector and are not modulatable (no input port). Each band’s Q controls its bandwidth:

Parameter Default Range Effect
q1–q10 2.0 0.1–10.0 Narrow (high Q) or wide (low Q) band influence

Low Q (0.5–1.0): Wide band — affects a broad frequency range around the center Default Q (2.0): One-third octave bandwidth — standard graphic EQ behavior High Q (5.0–10.0): Narrow band — precise surgical cuts or notches

Gain Values

Gain dB Equivalent Effect
0.0 Silent Band completely removed
0.5 −6 dB Reduce band by half
1.0 0 dB (unity) Band passes through unchanged (default)
1.5 +3.5 dB Moderate boost
2.0 +6 dB Strong boost

Note: Because all 10 filters output simultaneously, setting multiple bands to 2.0 can produce significant overall level increase. Monitor output levels when applying multiple boosts.

Usage Patterns

Basic Tonal Sculpting

Shape a Harmonic Generator’s output before signal conversion:

[Harmonic Generator] → [Rolloff Processor] → [Spectrum To Signal] → signalIn
                                                                          ↓
                                                                    [10-Band EQ] → signalOut
                                                                     band4: 0.5 (reduce mud)
                                                                     band7: 1.4 (add presence)

Dynamic Formant Shaping (Envelope Modulation)

Animate band gains with an envelope for vowel-like evolution:

[Envelope Engine] → controlOut → band5 (500 Hz)
                  → controlOut → band6 (1 kHz)
                                 [10-Band EQ] → signalOut

As the envelope rises, the mid-presence bands swell, creating a natural swell from dark to bright.

LFO Filter Sweep

Rhythmically modulate adjacent bands for a wah-like effect:

[LFO] → valueOut → band5
                    band6  → [10-Band EQ] → signalOut

Wire the LFO to multiple adjacent bands. As the LFO sweeps, the emphasis moves across the frequency spectrum.

Frequency Mapper + EQ

Pitch-adaptive EQ — high notes get more brightness:

[Frequency Mapper] → controlOut → band8 (4 kHz)
  outputMin: 0.8, outputMax: 1.8

Low notes get subtle 4 kHz content (0.8); high notes get a strong 4 kHz boost (1.8).

Sub Boost for Low Notes

[Frequency Mapper] → controlOut → band1 (31 Hz)
  curveType: Logarithmic
  invert: true
  outputMin: 0.8, outputMax: 1.6

Low pitched notes get a sub boost (up to 1.6×); high notes leave the sub band nearly untouched.

Carving Space Between Layers

When layering two signals via Signal Mixer, use two 10-Band EQs to carve complementary frequency spaces:

[Chain A] → [10-Band EQ (bands 1-5 = 1.0, bands 6-10 = 0.3)] → signalA → [Signal Mixer]
[Chain B] → [10-Band EQ (bands 1-5 = 0.3, bands 6-10 = 1.0)] → signalB → [Signal Mixer]

Chain A occupies the low end; Chain B occupies the high end.

Modulation Behavior

When a control signal is connected to a band port:

  1. The static parameter value is read as the baseline gain
  2. The connection function (passthrough, add, multiply, etc.) is applied with the incoming control value scaled to gain range: sourceValue = controlOut × 2.0
  3. The result is clamped to 0.0–2.0 before being applied to the filter

Q values have no input ports and cannot be modulated at runtime. Adjust them via the inspector.

Troubleshooting

No Output

  1. Check that signalIn is connected to a source producing audio
  2. Verify at least one band has a gain > 0.0

Unexpected Level Changes

  1. All 10 bands at unity gain should preserve level (within filter rounding error)
  2. If level is unexpectedly high, check for bands set > 1.0 via modulation
  3. The parallel architecture means the gain of each band is independent — silence a band (0.0) and the other 9 still contribute

Thin / Nasal Sound

  1. Try reducing Q values for the active bands (lower Q = wider, more natural sound)
  2. High Q on multiple adjacent bands can create a peaky, nasal coloration

Muddy Low End

  1. Reduce band4 (250 Hz) — this region is commonly associated with muddiness
  2. Reduce band3 (125 Hz) if the problem is in the low-mid warmth area

Too Much Sibilance

  1. Reduce band9 (8 kHz) and/or band8 (4 kHz)

Technical Notes

Filter Design

Each band uses a biquad bandpass filter implemented in Direct Form I (transposed). The coefficient calculation follows the Audio EQ Cookbook formula for a constant-peak bandpass filter (peak gain = 0 dB for Q computation).

The ten filters are independent — they all receive the same input signal and their outputs are summed. This is a true parallel graphic EQ architecture, not a series chain.

Filter State and Legato

The filter states (delay line history) are preserved across legato note transitions and reset at the start of non-legato notes. This prevents clicks and artifacts when notes connect smoothly.

CPU Cost

Ten biquad filter evaluations per sample (ten multiply-accumulate chains of 5 operations each), plus ten multiply-adds for the output sum. Moderate CPU cost — more expensive than single-filter containers but efficient relative to convolution-based approaches.
