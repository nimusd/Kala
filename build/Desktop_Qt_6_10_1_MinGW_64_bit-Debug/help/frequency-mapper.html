Frequency Mapper Documentation

Overview

The Frequency Mapper is a pitch-to-control converter that translates the current note’s fundamental frequency (in Hz) into a normalized control value. It is the bridge between the pitched world of a note and the control-rate world of modulatable parameters — letting pitch drive filter cutoffs, grain density, reverb mix, or any other modulatable input.

Four transfer curves are available (Linear, Logarithmic, Exponential, S-Curve), the output range is freely configurable, and an optional external curve input can override the built-in shaping entirely.

Category

Modifiers (Green) - Generates control signals from note properties

Core Concept

pitch (Hz) → normalize to [0, 1] → apply curve → scale to [outputMin, outputMax] → controlOut

The note’s fundamental pitch is automatically available — no pitch input port is needed. The mapper clamps pitches outside [lowFreq, highFreq] to the boundary values before normalizing.

Ports

Input Ports

Port Data Type Default Description
curve Control — Optional external curve value (0.0–1.0). When connected, replaces the built-in curve shape entirely
pitchMultiplier Control 1.0 Scales the note frequency before it is mapped. 1.0 = no change, 2.0 = octave up, 0.5 = octave down. Shifts the apparent pitch that the mapper sees, allowing transposed or vibrato-driven mapping without changing the score.

Note: The note pitch (Hz) is read internally from the audio graph context. The pitchMultiplier port scales this pitch before mapping — useful to shift the operating range or add vibrato to the mapped output.

Output Ports

Port Data Type Description
controlOut Control Mapped control value in [outputMin, outputMax]

Parameters

Parameter Type Default Range Description
lowFreq Float 25.0 1.0–20000.0 Frequency that maps to 0.0 (before output scaling)
highFreq Float 8000.0 lowFreq+1 – 20000.0 Frequency that maps to 1.0 (before output scaling)
outputMin Float 0.0 Any Minimum output value (reached at lowFreq)
outputMax Float 1.0 Any Maximum output value (reached at highFreq)
curveType Integer 1 (Logarithmic) 0–3 Transfer curve shape (see Curve Types)
invert Bool false — When true, flips the normalized value (t → 1-t) before scaling
pitchMultiplier Float 1.0 0.25–4.0 Pitch ratio applied before mapping (1.0 = no change, 2.0 = octave up, 0.5 = octave down)

Curve Types

0: Linear

output
 1 |          /
   |         /
   |        /
   |       /
   |      /
   |     /
 0 +----/------> pitch
       low  high
  • Formula: t = (pitch - lowFreq) / (highFreq - lowFreq) (linear space)
  • Use: Direct proportional mapping; equal Hz steps produce equal output steps
  • Character: Emphasizes the upper frequency range perceptually

1: Logarithmic (default)

output
 1 |       ╭────
   |      ╱
   |    ╱
   |  ╱
   |╱
 0 +-----------> pitch
     low      high
  • Formula: t = (log(pitch) - log(lowFreq)) / (log(highFreq) - log(lowFreq))
  • Use: Perceptually uniform mapping — octaves take equal output space
  • Character: Natural-sounding; matches human pitch perception
  • Best for: Filter cutoffs, spectral controls, any pitch-proportional modulation

2: Exponential

output
 1 |            /
   |           /
   |         ╱
   |       ╱
   |    ╱
   |╱
 0 +-----------> pitch (log scale)
     low      high
  • Formula: t = ((pitch - lowFreq) / (highFreq - lowFreq))² (applied after linear normalize)
  • Use: Accelerating response — low pitches produce very little output change; high pitches produce large change
  • Character: Compresses the low register, expands the high register

3: S-Curve

output
 1 |         ╭──
   |        ╱
   |       ╱
   |      ╱
   |   ╱
   |──╯
 0 +-----------> pitch
     low      high
  • Formula: t = t² × (3 - 2t) (Hermite smooth step)
  • Use: Gentle response at extremes, steeper in the middle
  • Character: Natural, organic mapping; avoids abrupt jumps at the frequency boundaries

Using the invert Parameter

When invert is enabled, the normalized t value is flipped before scaling: - Low pitches → high output values - High pitches → low output values

Example: With outputMin = 0.0, outputMax = 1.0, and invert = true, a low note produces controlOut ≈ 1.0 and a high note produces controlOut ≈ 0.0. Useful for filter cutoffs that should be bright for low notes and dark for high ones.

Using the External curve Input

When a control source is connected to the curve input port, the built-in curve type is bypassed entirely. The connected value (0.0–1.0) is used directly as the shaped t value before output scaling.

This allows: - Using an Easing Applicator or Envelope Engine to define a custom pitch-to-control transfer - Dynamically changing the curve shape by modulating the curve input - Sharing a single externally computed curve across multiple Frequency Mappers

Note: The pitch normalization step still runs (clamping pitch to [lowFreq, highFreq]) but the resulting normalized t is discarded in favour of the external value.

Output Range

The outputMin and outputMax parameters freely set the output range. They can be any values, including negative ranges and inverted ranges:

outputMin outputMax Effect
0.0 1.0 Standard 0–1 range
0.0 2.0 Gain control (0 to double)
-1.0 1.0 Bipolar modulation
1.0 0.0 Inverted (same as invert=true with 0–1 range)
200.0 8000.0 Direct Hz output for filter cutoff

Usage Patterns

Pitch-Tracking Filter Cutoff

Make a low-pass filter track the fundamental pitch:

[Harmonic Generator] → [Rolloff Processor] → [LP/HP Filter] → signalOut
                                                    ↑
                                           [Frequency Mapper] → controlOut → cutoff
                                            curveType: Logarithmic
                                            lowFreq: 25, highFreq: 4000
                                            outputMin: 300, outputMax: 6000

Low notes get a 300 Hz cutoff; high notes get a 6 kHz cutoff — automatically tracking the fundamental.

Brightness-by-Pitch (Rolloff Modulation)

Brighter sound for higher pitches:

[Frequency Mapper] → controlOut → [Rolloff Processor] (lowRolloff)
  curveType: Logarithmic
  outputMin: 0.2 (bright)
  outputMax: 0.8 (rolled off)
  invert: true

With invert, low pitches get a high rolloff value (more harmonics), high pitches get less rolloff.

Reverb Amount by Register

More reverb on high notes:

[IR Convolution] ← wetDry ← [Frequency Mapper]
                              lowFreq: 100, highFreq: 2000
                              outputMin: 0.1, outputMax: 0.9

Drift/LFO Depth by Pitch

Wider drift for high pitched sounds:

[Frequency Mapper] → controlOut → [Drift Engine] (depth parameter via modulation)
  curveType: Exponential
  outputMin: 0.01, outputMax: 0.3

Custom Curve (Easing Applicator)

Use an easing curve to define a completely custom pitch-to-control mapping:

[Easing Applicator] → easedValue → curve → [Frequency Mapper] → controlOut
  progress: ??? → use Frequency Mapper pitch to drive Easing's progress

In practice, connect a second Frequency Mapper’s controlOut (with Linear curve, 0–1 range) to an Easing Applicator’s progress input, then connect the Easing output to this Frequency Mapper’s curve input.

Parameter Effects Reference

lowFreq / highFreq

Defines the pitch range over which the mapping runs. Pitches below lowFreq clamp to outputMin (or outputMax if inverted); pitches above highFreq clamp to outputMax.

lowFreq highFreq Pitch Range
25 8000 Full practical voice range (default)
55 880 Two octaves around A standard
100 2000 Mid-register focus
20 20000 Full audible spectrum

Troubleshooting

Output Never Changes

  1. Verify the note’s pitch falls within [lowFreq, highFreq] — notes outside this range clamp to the boundary value
  2. Check that outputMin ≠ outputMax (zero range produces a constant output)
  3. Confirm no external curve connection is overriding with a constant value

Output Too Abrupt

  1. Switch to Logarithmic or S-Curve for smoother perceptual response
  2. Narrow the [lowFreq, highFreq] range to focus on the pitches in use

Output Too Subtle

  1. Widen the [outputMin, outputMax] range
  2. Enable invert if the effect seems backwards

Technical Notes

Stateless Processing

The Frequency Mapper has no internal state. The output at any sample depends only on the current pitch and parameter values. The reset() method is a no-op included for API consistency.

Pitch Source

Pitch is provided by the audio graph’s generateSample(pitch, ...) call — the same pitch that drives all other pitch-sensitive containers. It reflects the note’s instantaneous fundamental frequency (including any modulation applied upstream).

External Curve Override

When curve is connected, the full map(pitch, externalCurve) overload is used. The pitch normalization still runs (to clamp pitches at the boundaries), but the external value completely replaces the built-in curve result before the invert and output-range scaling steps.
