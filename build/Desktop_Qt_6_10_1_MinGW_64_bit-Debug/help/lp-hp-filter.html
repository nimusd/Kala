LP/HP Filter Documentation

Overview

The LP/HP Filter is a biquad low-pass or high-pass filter with modulatable cutoff and resonance. It is the simplest and most direct brightness-shaping tool in the Filters category: low-pass mode removes high-frequency content above the cutoff; high-pass mode removes low-frequency content below it. The resonance parameter adds a peak at the cutoff frequency for more pronounced filtering character.

Filter Modes

0: Low-Pass

Passes frequencies below the cutoff and attenuates frequencies above it. The attenuation slope is −12 dB/octave (second-order filter).

Use for: - Removing harshness and high-frequency noise from a signal - Creating warm, dark, or muffled textures - Smoothing the top end of a bright harmonic chain - Pitch-tracking filter for brightness-by-pitch (via Frequency Mapper)

Resonance (Q Factor)

The resonance parameter is the filter’s Q factor — higher values create a sharper peak at the cutoff frequency.

Resonance Character
0.5 Overdamped — very gentle rolloff, no peak
0.707 (default) Butterworth — maximally flat, no resonant peak
1.0–2.0 Mild resonance — slight emphasis at cutoff
3.0–5.0 Noticeable resonance peak — classic synth filter character
8.0–15.0 Strong resonance — pronounced whistle at cutoff
18.0–20.0 Near-self-oscillation — extreme ringing at cutoff

0.707 (Butterworth Q) is the default and provides the flattest passband with no overshoot. For classic synthesizer filter sweeps, increase resonance toward 2–5.

Cutoff Frequency Reference

Usage Patterns

Basic Low-Pass Brightness Control

[Spectrum To Signal] → signalIn → [LP/HP Filter] → signalOut
                                    mode: LowPass
                                    cutoff: 2000 Hz
                                    resonance: 0.707

Rolls off the top end of a harmonic chain for a warmer character.

Envelope-Controlled Filter Sweep

Classic envelope-to-filter sweep:

[Envelope Engine] → controlOut → cutoff → [LP/HP Filter] → signalOut
                                            mode: LowPass
                                            resonance: 2.5

The envelope opens the filter from dark (low cutoff) to bright (high cutoff) over the note’s duration. Wire the Envelope’s output via a connection with multiply function and a high weight to scale 0–1 up to 0–20000.

Frequency Mapper Pitch-Tracking Filter

Automatically adjust cutoff based on the note’s pitch:

[Frequency Mapper] → controlOut → cutoff
  outputMin: 500, outputMax: 8000
  curveType: Logarithmic
                         [LP/HP Filter] → signalOut
                          mode: LowPass
                          resonance: 1.5

Low notes get a 500 Hz cutoff (dark); high notes get an 8 kHz cutoff (bright).

LFO Filter Wobble

[LFO] → valueOut → cutoff
                     [LP/HP Filter] → signalOut
                      mode: LowPass
                      cutoff: 1000 Hz (baseline)
                      resonance: 3.0

The LFO modulates the cutoff for a rhythmic filter wobble effect.

High-Pass for Clarity / Thinning

[Signal Source] → signalIn → [LP/HP Filter] → signalOut
                               mode: HighPass
                               cutoff: 200 Hz
                               resonance: 0.707

Removes low-frequency mud without affecting the midrange. Useful when layering multiple sounds.

Resonant Emphasis at a Frequency

Set cutoff to a specific frequency and increase resonance to add a tonal emphasis:

mode: LowPass
cutoff: 800 Hz
resonance: 6.0

This creates a pronounced peak at 800 Hz while attenuating above it — similar to a tuned bandpass for that region.

Serial Filtering (High-Pass then Low-Pass)

To create a bandpass effect, chain two LP/HP Filters in series:

[Signal] → [HP Filter, cutoff: 300 Hz] → [LP Filter, cutoff: 3000 Hz] → [Output]

This passes only the 300 Hz – 3 kHz band, eliminating both sub and ultra-high content.

Modulation Behavior

When control sources are connected to cutoff or resonance:

  • cutoff: Result is clamped to [20, 20000] Hz after the connection function is applied
  • resonance: Result is clamped to [0.5, 20.0] after the connection function is applied

mode has no input port and must be set via the inspector. Changing mode while notes are playing may produce a click (due to filter state mismatch); prefer setting mode before playback.

The LP/HP Filter differs from the Rolloff Processor in that it operates in the signal domain (after Spectrum To Signal) and can be placed anywhere in a signal chain, including after Formant Body or Breath Turbulence.

Troubleshooting

No Audible Effect

  1. Confirm signalIn is connected
  2. Check that the cutoff is set to a frequency that actually affects the signal’s content (cutting at 20 kHz on a signal with no high-frequency content will have no audible effect)
  3. Verify the filter mode matches the desired effect

Harsh Resonance Peak

  1. Reduce resonance toward 0.707–1.0
  2. If using high resonance with a modulatable cutoff, be aware that sweeping through resonance at high Q values produces a pronounced “whistle” — this is expected

Clicks When Mode Changes

The filter’s internal state (biquad delay history) is built up for one mode. Switching mode changes the coefficient calculation, causing a brief discontinuity. Avoid mode changes during note playback; set mode before starting audio.

Low Output Level

The LP/HP Filter at unity has approximately 0 dB passband gain. However, strong resonance (Q > 5) can cause high-gain peaks at the cutoff that may cause downstream clipping. Reduce resonance or add a gain stage after the filter.

Technical Notes

Filter Design

Uses Audio EQ Cookbook coefficients for biquad lowpass and highpass. The implementation uses Direct Form I (transposed) for numerical stability.

The transfer function (z-domain):

H(z) = (b0 + b1·z⁻¹ + b2·z⁻²) / (1 + a1·z⁻¹ + a2·z⁻²)

Coefficients are recomputed whenever cutoff or resonance change.

Reset Behavior

Filter state (x1, x2, y1, y2 delay values) is zeroed at note start for non-legato notes. Preserved for legato transitions to prevent clicks at note boundaries.

CPU Cost

Single biquad — 5 multiply-accumulate operations per sample. One of the cheapest filter implementations available. Multiple LP/HP filters in a graph are effectively free compared to IR Convolution.
