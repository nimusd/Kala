Drift Engine Documentation

Overview

The Drift Engine is a control signal generator that produces slow-moving detuning multipliers for organic, living sound quality. It outputs values oscillating around 1.0 (e.g., 0.995 to 1.005), which can be multiplied with pitch or other parameters to create subtle beating and micro-variations.

Unlike a signal processor, the Drift Engine doesn’t take audio input - it generates control values that you connect to modulation targets elsewhere in your graph.

Category

Modifiers (Green) - Generates time-varying control signals

Core Concept

output = 1.0 + (driftSignal × amount)

The drift signal oscillates between -1 and +1, so the output ranges from (1.0 - amount) to (1.0 + amount). When multiplied with a frequency, this creates detuning.

Example: With amount = 0.005, output ranges from 0.995 to 1.005. A 440 Hz pitch becomes 437.8 Hz to 442.2 Hz - subtle but audible as organic movement.

Ports

Input Ports

Port Data Type Default Description
amount Control 0.005 Drift intensity (0.0 = none, 0.1 = extreme)
rate Control 0.5 Drift speed in Hz (how fast it wanders)

Output Ports

Port Data Type Description
driftOut Control Detuning multiplier centered around 1.0

Note: There is no signal input port. The Drift Engine generates control values, not processed audio.

Parameters

Parameter Type Default Range Description
amount Float 0.005 0.0 - 0.1 Drift intensity (deviation from 1.0)
rate Float 0.5 0.01 - 11.0 Drift oscillation speed in Hz
driftPattern Integer 2 0-2 Oscillation pattern (see below)

Drift Patterns

The driftPattern parameter selects how the drift oscillates:

ID Pattern Description
0 Sine Simple sine wave LFO - smooth, predictable
1 Random Sample-and-hold random values - stepped, unpredictable
2 Perlin Simulated Perlin noise (multiple layered sines) - smooth, organic (default)

Perlin Pattern Details

The Perlin pattern combines three sine waves at related but irrational frequency ratios (1.0, 0.83, 1.27) with decreasing amplitudes (1.0, 0.5, 0.25). This creates smooth, non-repeating drift that sounds more natural than a single sine wave.

Usage Patterns

Basic Pitch Drift

Connect drift output to modulate pitch for organic detuning.

Drift Engine ──→ driftOut ──→ [multiply with pitch]
     │
     └── amount: 0.005, rate: 0.3

The drift output (around 1.0) is multiplied with pitch frequency to create micro-detuning.

Harmonic Generator Detuning

Apply drift to the purity or drift input of a Harmonic Generator.

Drift Engine ──→ driftOut ──→ Harmonic Generator (drift input)

Ensemble Effect (Multiple Drift Engines)

Use several Drift Engines with different rates for chorus-like beating.

Drift Engine 1 (rate: 0.3) ──→ Voice 1 pitch
Drift Engine 2 (rate: 0.5) ──→ Voice 2 pitch
Drift Engine 3 (rate: 0.7) ──→ Voice 3 pitch

The slightly different drift rates create natural beating patterns like an acoustic ensemble.

Filter Modulation

Connect drift to filter parameters for subtle timbral movement.

Drift Engine ──→ driftOut ──→ Rolloff Processor (lowRolloff)
     │
     └── amount: 0.02, rate: 0.8

Dynamic Drift Amount

Modulate the drift amount from another source for evolving intensity.

Envelope Engine ──→ Drift Engine (amount input)
                          │
                          └──→ driftOut ──→ [target]

Important Behaviors

Output Range

The output is always centered around 1.0: - Minimum: 1.0 - amount - Maximum: 1.0 + amount

With default amount (0.005): output ranges from 0.995 to 1.005.

Multiplicative Use

The drift output is designed to be multiplied with frequencies, not added:

effectivePitch = basePitch × driftOut

This maintains musical intervals regardless of the base frequency.

Stateful Operation

Unlike the Easing Applicator, the Drift Engine is stateful - it maintains internal phase counters that advance each sample. The output depends on the history of processing, not just current inputs.

Reset Behavior

Calling reset() returns all phase counters to zero, restarting the drift pattern from the beginning.

Technical Notes

Implementation

The Drift Engine generates samples using phase accumulators:

// Sine pattern
driftValue = sin(phase);
phase += 2π × rate / sampleRate;

// Convert to multiplier
output = 1.0 + (driftValue × amount);

Perlin Noise Simulation

The Perlin pattern uses three sine waves with irrational frequency ratios: - Primary: rate × 1.0 - Secondary: rate × 0.83, amplitude 0.5 - Tertiary: rate × 1.27, amplitude 0.25

The sum is normalized to stay within -1 to +1.

Phase Wrapping

Phases wrap at 2π × 1000 to prevent floating-point precision loss over long playback times.

Performance

  • Lightweight trigonometric operations
  • Minimal state (3 phase values + random state)
  • Efficient for many simultaneous instances
