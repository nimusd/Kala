Kala Dialog Documentation

This document describes the UI interaction of all dialogs in Kala.

1. New Project Dialog

Window Title: New project Size: 400 x 300 Modal: Yes Source: dialogNewProject.ui Background Color: Light green (rgb(220, 232, 214))

How to Open

Triggered from the File menu when creating a new project.

UI Components

Component Type Description
Project name QLineEdit Text field for entering the new project name
OK Button Accepts the dialog and creates the project with the entered name
Cancel Button Dismisses the dialog without creating a project

Interaction Flow

  1. User opens the dialog from the File menu.
  2. User types a project name into the text field.
  3. User clicks OK to create the project, or Cancel to abort.

2. Scale Dialog

Window Title: Scale Settings Size: 400 x 200 Modal: Yes Source: scaledialog.h, scaledialog.cpp, scaledialog.ui

How to Open

Click the Edit button in the Scale panel at the bottom of the Composition tab. The Scale panel displays the current scale in the format 1 · [Scale Name] · [Base Freq] Hz.

UI Components

Component Type Description
Scale QComboBox Dropdown populated with all available scales (20+ world scales). Each entry shows the scale name and stores the scale ID as user data.
Base Frequency QDoubleSpinBox Sets the reference frequency. Range: 10.00 - 1000.00 Hz. Default: 25.00 Hz. Suffix: ” Hz”.
OK Button Accepts the selected scale and base frequency
Cancel Button Dismisses without changing the scale

Interaction Flow

  1. Dialog opens pre-populated with the current scale selected in the dropdown and the current base frequency in the spinbox.
  2. User selects a different scale from the dropdown (e.g., Pythagorean, Just Intonation, Raga Bhairav, etc.).
  3. User optionally adjusts the base frequency.
  4. Clicking OK applies the new scale and base frequency to the ScoreCanvas. The canvas regenerates its scale lines using the new tuning ratios.
  5. Clicking Cancel discards any changes.

Data Flow

  • On accept, the caller retrieves the selected scale via getSelectedScale() and base frequency via getBaseFrequency().
  • The ScoreCanvas then calls setScale() and setBaseFrequency(), which regenerate the colored horizontal scale lines across octaves 0-9.

3. Composition Settings Dialog

Window Title: Composition Settings Size: 400 x 450 Modal: Yes Source: compositionsettingsdialog.h, compositionsettingsdialog.cpp

How to Open

Accessible via File > Composition Settings. Syncs with ScoreCanvasWindow toolbar controls.

UI Components

Component Type Description
Name QLineEdit The composition name
Time Mode Two QRadioButtons Absolute or Musical. Determines whether the composition length is specified in clock time or in bars.
Tempo QSpinBox Range: 20 - 300 BPM. Suffix: ” BPM”.
Time Signature Two QSpinBoxes Numerator (1-99) / Denominator (0-99), displayed side by side with a “/” separator.
Length (Absolute) QTimeEdit Displayed when Absolute mode is selected. Format: HH:mm:ss.
Length (Musical) QSpinBox Displayed when Musical mode is selected. Range: 1 - 9999. Suffix: ” bars”.
Calculated Info QGroupBox Read-only section showing computed values: bar duration in ms and total duration in bars = MM:SS.mmm format.
OK Button Accepts settings
Cancel Button Dismisses without saving

Interaction Flow

  1. Dialog opens pre-populated with current composition settings.
  2. User can change the composition name.
  3. Selecting a Time Mode radio button switches the Length input:
    • Absolute: Shows the HH:mm:ss time editor.
    • Musical: Shows the bars spinbox.
  4. Changing Tempo, Time Signature, or Length values immediately updates the “Calculated Info” panel, showing:
    • Bar duration in milliseconds
    • Total duration in both bars and MM:SS.mmm format
  5. When the user changes duration in Absolute mode, the bars count is auto-synced (and vice versa). Signal blocking prevents infinite update loops.
  6. Clicking OK returns the edited CompositionSettings struct via getSettings().

Live Updates

All input fields are connected to recalculation slots. Any change to tempo, time signature, duration, or bar count triggers updateLengthConversion(), which refreshes the Calculated Info labels in real time.

4. Go To Dialog

Window Title: Go To Size: 350 x 200 Modal: Yes Source: gotodialog.h, gotodialog.cpp

How to Open

Triggered by pressing the G keyboard shortcut in the Composition tab.

The dialog adapts its layout based on the current time mode:

Musical Mode (Measure:Beat:MS):

Component Type Description

Common Components:

Component Type Description
Mode label QLabel Shows “Go to position (Minutes:Seconds:Milliseconds):” or “Go to position (Measure:Beat:Milliseconds):”
Info label QLabel Gray italic text showing the calculated target time in ms (and in MM:SS.mmm format for Musical mode)
OK Button Navigates to the target position
Cancel Button Dismisses without navigating

Interaction Flow

  1. Dialog opens with the appropriate input mode based on the composition’s current time mode.
  2. The first input field receives focus with its value selected for quick typing.
  3. As the user changes any value, the info label updates in real time showing the calculated target time in milliseconds.
  4. In Musical mode, the calculation uses: (bar-1) * barDuration + (beat-1) * beatDuration + ms.
  5. Clicking OK returns the target time via getTargetTimeMs(), and the ScoreCanvas scrolls to that position.

The Parameters group shows/hides controls based on the selected easing category:

Interaction Flow

  1. User selects an easing type from the categorized dropdown.
  2. The curve preview updates to show the easing shape.
  3. Relevant parameter sliders appear. Each slider is bidirectionally synced with its spinbox.
  4. Adjusting any parameter immediately updates the curve preview.
  5. User selects an anchor mode to pin certain notes in place.
  6. User adjusts the weight to control effect intensity (1.0 = normal, <1.0 = subtle, >1.0 = exaggerated).
  7. Clicking OK returns the configured Easing object, anchor mode, and weight to the caller.

Digit Formula Sub-Panel

Appears below the Sequence dropdown when Digit Formula is selected. The core idea: each fractional decimal digit (0–9) of the chosen number maps to a harmonic amplitude of digit / 9.0. The digits themselves provide a natural 0–1 normalization with no additional scaling needed.

ComponentTypeDescription
Constant QComboBox Select a built-in constant (π, e, φ, √2, √3, √5 — each with 120 hardcoded digits) or Custom formula to enter A and B manually.
Formula A / op / B QDoubleSpinBox + QComboBox Visible when Custom formula is selected. Operators: ÷, ×, +, −, √A (unary), A² (unary), Aᴮ (power). B is hidden for unary operators. Rational operations (÷, ×, +, −, A², integer Aᴮ) use exact __int128 long division — no digit limit. Irrational operations (√A, non-integer Aᴮ) use long double (~18 digits).
Start digit QSpinBox Offset into the fractional expansion. Range: 0–119 for constants, 0–9999 for rational formulas, 0–17 for irrational. Sliding this explores different "regions" of the number — each window position produces a distinct spectrum from the same formula.
Normalize amplitude QCheckBox When checked (default), normalizes total amplitude so all patterns have equal loudness. When unchecked, preserves raw digit/9 ratios — passages with many zeros are quieter.
Result label QLabel Shows the active digit window: digits [0 – 63]: 14159265358979323846264338… (120 available, 64 harmonics)

Rational vs Irrational

  • Rational formulas (1÷7, 788÷1.36, etc.) produce repeating decimals. The period length is related to the denominator — prime denominators give the longest periods. Digits are computed by exact integer long division, so the window can reach offset 9999 with perfect accuracy.
  • Irrational constants (π, e, φ…) produce non-repeating sequences. Each window position is unique. These are hardcoded to 120 digits.
  • Fractions with prime denominators (1/7, 1/17, 1/97…) are the most musically interesting: the period length divides p−1, so 1/97 has a 96-digit period — longer than 64 harmonics, effectively sounding non-repeating.

Window Offset as a Performance Parameter

The Start digit set in this dialog becomes the base value of the digitWindowOffset container parameter. This parameter has a corresponding input port on the Harmonic Generator, so it can be driven by a score curve via an Envelope Engine. As the offset changes during playback, harmonic amplitudes update in real time — the timbre morphs continuously through the mathematical landscape of the formula. See the Harmonic Generator documentation for the full setup.

Interaction Flow

  1. Dialog opens with 64 harmonics initialized to the “All Harmonics” sequence.
  2. Selecting a sequence generates a new pattern and displays it in the spectrum visualizer. All generated patterns are normalized so amplitudes sum to 1.0.
  3. Editing the spectrum directly: drag individual bar tops up or down. Double-click a bar to restore its generated value. Press Delete to zero out a bar. Changes feed back into the internal customPattern array.
  4. Selecting “Custom” preserves the current pattern and allows free-form editing without regenerating.
  5. Save DNA opens a file dialog for .dna.json files. Load DNA loads a file and switches the dropdown to “Custom”.
  6. Clicking OK returns the pattern via getCustomDnaPattern().
  7. If setCustomPattern() was called before showing the dialog (editing an existing DNA), the dialog opens with that pattern displayed and the dropdown set to “Custom”.

File Format

{
  "name": "Custom DNA Pattern",
  "description": "Custom harmonic distribution created in Kala DNA Editor",
  "version": "1.0",
  "numHarmonics": 64,
  "amplitudes": [1.0, 0.5, 0.33, ...]
}

Interaction Flow

  3. New creates a new user envelope with a unique name (“New Envelope”, “New Envelope 2”, etc.) initialized as a linear rise. It is immediately added to the preset list and selected for editing.

Shared UI Patterns

EnvelopeCurveCanvas

Several dialogs share the EnvelopeCurveCanvas widget for interactive curve editing: - Vibrato Editor (intensity envelope) - Dynamics Curve (dynamics shape) - Envelope Library (general-purpose envelopes)

Common interactions: - Click on empty area to add a control point - Drag a point to reposition it - Right-click a point to delete it - Shift+drag to snap to grid

Slider-Spinbox Pairs

The Easing Dialog and Vibrato Editor use paired slider/spinbox controls where adjusting one automatically updates the other. The sliders use integer values scaled by 10x or 100x to represent decimal ranges.

Preset Systems

Three dialogs include preset systems: - Vibrato Editor: Factory + user presets (persisted to JSON in app data) - Dynamics Curve: Built-in presets + file save/load - Envelope Library: Factory presets (read-only) + session-only user presets + file save/load

File Persistence

File-based save/load uses .env.json (envelopes/dynamics) and .dna.json (harmonic patterns) formats. All file dialogs remember the last used directory via QSettings.
