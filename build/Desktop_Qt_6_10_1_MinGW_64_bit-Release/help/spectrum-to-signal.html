Spectrum to Signal Documentation

Overview

Spectrum to Signal is an additive synthesis engine that converts harmonic spectrum data into an audio signal. It maintains phase accumulators for each harmonic and sums sine waves at the specified pitch and its harmonics.

This is an Essential container - every sound-producing graph must include exactly one Spectrum to Signal to generate audible output.

Category

Essential (Blue) - Required for sound generation

Core Concept

For each harmonic h (1, 2, 3, ...):
    signalOut += amplitude[h] × sin(phase[h])
    phase[h] += 2π × pitch × h / sampleRate

The spectrum provides amplitude for each harmonic. The container maintains phase continuity across samples, generating smooth sine waves that sum to the final audio output.

Ports

Input Ports

Port Data Type Default Description
spectrumIn Spectrum (empty) Harmonic amplitudes from upstream (Harmonic Generator, Rolloff Processor)
pitchMultiplier Control 1.0 Scales the note frequency before synthesis. 1.0 = no change, 2.0 = octave up, 0.5 = octave down. Connect LFO for vibrato, Envelope Engine for pitch glide.
normalize Control 1.0 Normalization blend (see below)

Output Ports

Port Data Type Description
signalOut Signal Summed audio output (-1.0 to +1.0 range, ideally)

Parameters

Parameter Type Default Range Description
normalize Float 1.0 0.0-1.0 Controls amplitude normalization behavior
pitchMultiplier Float 1.0 0.25–4.0 Pitch ratio applied before synthesis (1.0 = no change, 2.0 = octave up, 0.5 = octave down). Overridden by connection to pitchMultiplier port.

Normalization Behavior

The normalize parameter controls how the summed output is scaled:

if (normalize > 0.0) {
    output /= (totalAmplitude × (1.0 - normalize) + normalize)
}
normalize Divisor Effect
0.0 (skipped) Raw sum, no scaling - may clip with many harmonics
0.5 blend Partial normalization
1.0 1.0 Divides by 1.0 - effectively raw sum

Practical guidance: - For controlled input spectra (already normalized): use normalize = 1.0 - For unknown/variable spectra: experiment with values around 0.3-0.7 - The default (1.0) passes through the raw sum

Pitch Multiplier

The pitchMultiplier port and parameter scale the note's fundamental frequency before synthesis:

effectivePitch = notePitch × pitchMultiplier

The static slider value is used when the port is unconnected. Connecting a modulator overrides the static value. Use this for vibrato (LFO), pitch glide (Envelope Engine), spring pitch wobble (Physics System), or static transposition (inspector slider).

Phase Management

Phase Accumulators

The container maintains a phase accumulator for each harmonic (up to 64 by default). Phases advance continuously:

phases[h] += 2π × pitch × harmonicNumber / sampleRate

Phase Wrapping

Phases wrap at 2π × 1000 to prevent floating-point precision loss during long playback.

Reset

reset() zeros all phase accumulators. This is called at note start to ensure consistent attack transients.

Usage Patterns

Minimal Sound Chain

The simplest possible sound-producing graph:

Harmonic Generator ──→ spectrum ──→ Spectrum to Signal ──→ signalOut ──→ [Audio Output]

With Spectrum Shaping

Add rolloff processing between generator and converter:

Harmonic Generator ──→ Rolloff Processor ──→ Spectrum to Signal ──→ [Audio Output]

With Formant Processing

Process the signal after conversion:

Harmonic Generator ──→ Spectrum to Signal ──→ Formant Body ──→ [Audio Output]

Pitch Modulation

Add vibrato or other pitch effects via the pitchMultiplier port:

LFO ──→ pitchMultiplier
                │
Harmonic Generator ──→ Spectrum to Signal ──→ [Audio Output]

Multiple Spectrum Sources

Only one Spectrum to Signal can exist, but multiple containers can feed into it through a chain:

Harmonic Generator ──→ Rolloff 1 ──→ Rolloff 2 ──→ Spectrum to Signal

Comparison with Other Containers

Container Input Output Role
Harmonic Generator (parameters) Spectrum Creates harmonic content
Rolloff Processor Spectrum Spectrum Shapes harmonic amplitudes
Spectrum to Signal Spectrum Signal Converts to audio
Formant Body Signal Signal Shapes audio with resonances

Spectrum to Signal is the bridge between the spectrum domain (harmonic amplitudes) and the signal domain (audio waveform).

Important Behaviors

Harmonic Skipping

Harmonics with zero amplitude are skipped (no CPU spent on silent components).

Additive Nature

The output is a pure sum of sine waves. This creates: - No intermodulation distortion - Linear phase response - Predictable harmonic content

Amplitude Accumulation

With many harmonics at significant amplitudes, the raw sum can exceed ±1.0. Consider: - Using normalized spectra from Harmonic Generator - Applying gain reduction downstream - Experimenting with the normalize parameter

Technical Notes

Additive Synthesis Formula

The output at time t is:

signal(t) = Σ [A_n × sin(2π × f × n × t + φ_n)]

Where: - A_n = amplitude of harmonic n - f = fundamental frequency (pitch) - n = harmonic number (1, 2, 3, …) - φ_n = accumulated phase for harmonic n

CPU Considerations

Processing cost scales with active harmonics: - 64 harmonics = 64 sine calculations per sample - At 44.1kHz, that’s ~2.8 million sine calls per second - Harmonics with zero amplitude are skipped

Nyquist Considerations

The current implementation does not filter harmonics above Nyquist. With high pitches and many harmonics, aliasing may occur. For example: - Pitch = 1000 Hz, 64 harmonics → highest = 64000 Hz - At 44.1kHz sample rate, harmonics above 22050 Hz will alias

For high-pitched content, consider using fewer harmonics or accepting some aliasing.
