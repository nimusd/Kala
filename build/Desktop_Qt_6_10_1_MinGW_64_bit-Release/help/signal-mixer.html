Signal Mixer Documentation

Overview

The Signal Mixer is a signal combining tool that takes two audio signal inputs with independent, modulatable gain controls and outputs their weighted sum. It is the missing building block for combining independent signal chains into a single output.

For three or more chains, cascade multiple Signal Mixers.

Category

Essential (Blue) - Core signal routing

Core Concept

output = signalA * gainA + signalB * gainB

Where signalA and signalB are audio-rate inputs (typically from containers with signalOut ports), and gainA/gainB are per-channel amplitude controls (default 1.0, modulatable range 0.0-2.0).

Ports

Input Ports

Port Data Type Default Description
signalA Signal 0.0 First audio signal source
signalB Signal 0.0 Second audio signal source
gainA Control 1.0 Gain multiplier for signal A (0.0-2.0)
gainB Control 1.0 Gain multiplier for signal B (0.0-2.0)

Output Ports

Port Data Type Description
signalOut Signal Weighted sum of both input signals

Parameters

Parameter Type Default Range Description
gainA Float 1.0 0.0-2.0 Static gain for channel A (overridden when modulated)
gainB Float 1.0 0.0-2.0 Static gain for channel B (overridden when modulated)

Gain Values

Gain Effect Use Case
0.0 Channel muted Solo the other channel
0.5 Half volume Subtle blend
1.0 Unity gain (default) Full signal, no attenuation
1.5 Boost Emphasize one channel
2.0 Maximum boost Strong emphasis (watch for clipping)

Usage Patterns

Basic Two-Chain Mix

Combine a Spectrum to Signal chain with a Karplus Strong chain:

Harmonic Generator --> Spectrum to Signal --signalOut--> signalA --+
                                                                    |
                                                          Signal Mixer --> [Output]
                                                                    |
        Noise Color Filter --> Karplus Strong --signalOut--> signalB --+

Envelope-Controlled Crossfade

Fade channel B in over the note’s duration:

Spectrum to Signal --signalOut--> signalA --+
                                             |
                                   Signal Mixer --> [Output]
                                             |
    Formant Body --signalOut-----> signalB --+
                                        ^
                  Envelope Engine --> gainB

Set gainA to 1.0 (static) and wire an Envelope Engine to gainB. As the envelope rises from 0.0 to 1.0, the second signal fades in.

LFO-Modulated Blend

Rhythmic pulsing between two signals:

Spectrum to Signal --signalOut--> signalA --+
                                             |
                                   Signal Mixer --> [Output]
                                             |
    Formant Body --signalOut-----> signalB --+
                                        ^
                             LFO --> gainB

A slow LFO (0.5-2 Hz) on gainB creates a pulsing effect. The gain is clamped to 0.0-2.0 so bipolar LFO output is safe.

Three-Way Mix (Cascaded)

Combine three independent signal chains:

Chain A --signalOut--> signalA --+
                                  |
                        Mixer 1 --signalOut--> signalA --+
                                  |                       |
Chain B --signalOut--> signalB --+              Mixer 2 --> [Output]
                                                          |
Chain C ----------------------signalOut--> signalB ------+

Set all gains to balance the three chains. For equal contribution from all three, set Mixer 1 gains to 1.0 and Mixer 2’s gainA to 0.67, gainB to 0.33 (or simply leave all at 1.0 and accept the louder sum).

Solo/Mute With Static Gains

Quick A/B comparison of two signal chains:

  • Solo A: Set gainA = 1.0, gainB = 0.0
  • Solo B: Set gainA = 0.0, gainB = 1.0
  • Equal mix: Set gainA = 1.0, gainB = 1.0
  • A dominant: Set gainA = 1.0, gainB = 0.3

Parallel Processing

Process one signal two different ways and combine:

                          +--> Formant Body --signalOut--> signalA --+
                          |                                          |
Spectrum to Signal --+----|                                Signal Mixer --> [Output]
                          |                                          |
                          +--> Karplus Strong --signalOut--> signalB --+

Practical Applications

Sound Design Techniques

1. Layered Timbres

  • Combine a bright harmonic chain with a filtered noise chain for textured pads
  • Mix a clean tone with a formant-processed version for vocal character

2. Attack Layering

  • Chain A: Sustained tone (Spectrum to Signal)
  • Chain B: Plucked attack (Karplus Strong) with envelope on gainB
  • Result: Pluck attack decays into sustained body

3. Noise Blending

  • Chain A: Clean pitched signal
  • Chain B: Noise Color Filter output
  • Use gainB to control noise floor amount

4. Dynamic Mixing

  • Wire separate Envelope Engines to gainA and gainB
  • Create crossfade effects, ducking, or swell patterns

Gain Amount Guidelines

  • Subtle layer: 0.1-0.3 - Background texture
  • Equal blend: 0.5-1.0 - Both chains prominent
  • Emphasis: 1.0-1.5 - Featured chain
  • Boost: 1.5-2.0 - Strong emphasis (monitor levels)

Troubleshooting

No Audio Output

  1. Verify at least one signalA or signalB port is connected
  2. Check that the connected source containers are producing audio
  3. Verify gain parameters are not both set to 0.0
  4. Ensure the Signal Mixer is the final output container (no outgoing signalOut connections)

Only Hearing One Channel

  1. Check that both signal input ports have valid connections
  2. Verify the silent channel’s gain is not 0.0
  3. Confirm the source container for the silent channel is working (test by connecting it directly as the final output)

Clipping or Distortion

  1. Reduce gain values - two full-amplitude signals summed at unity gain can exceed 1.0
  2. Try gainA = 0.7, gainB = 0.7 for headroom
  3. Avoid gain values above 1.5 unless one channel is quiet

Unexpected Volume

  1. Remember that gains default to 1.0 - summing two signals doubles peak amplitude
  2. For equal-loudness mixing, try gains of 0.5-0.7 per channel
  3. Use envelope modulation on gains for controlled dynamics

Important Behaviors

Signal Domain

The Signal Mixer operates in the signal domain (audio-rate samples), not the spectrum domain. Connect it to containers with signalOut ports (Spectrum to Signal, Formant Body, Breath Turbulence, Noise Color Filter, Karplus Strong, or another Signal Mixer).

Gain Clamping

When gain ports are modulated via connections, the resulting gain values are clamped to 0.0-2.0 after the connection function is applied. This prevents negative gain (phase inversion) and excessive boost.

No State

The Signal Mixer is stateless - the output depends only on the current sample’s inputs. This means: - No reset needed at note boundaries - No artifacts from abrupt gain changes - Deterministic output for given inputs - Negligible CPU cost (two multiplies and one add per sample)

Unconnected Inputs

If a signal input has no connection, it defaults to 0.0 (silence). This means: - A mixer with only signalA connected outputs signalA * gainA - A mixer with no signal connections outputs silence

Comparison with Other Containers

Container Domain Combining Method
Signal Mixer Signal Weighted sum of two audio signals
Spectrum Blender Spectrum Linear interpolation between two spectra
Breath Turbulence Signal Voice/noise crossfade blend
Formant Body Signal Resonant filter (single input)

The Signal Mixer differs from Breath Turbulence in that it performs a pure weighted sum (both signals can be at full volume simultaneously), whereas Breath Turbulence performs a crossfade (the two signals share a fixed total energy). The Signal Mixer also differs from the Spectrum Blender, which operates in the harmonic/spectrum domain before audio conversion.

Technical Notes

Processing Formula

signalOut = signalA * gainA + signalB * gainB

No normalization, limiting, or saturation is applied. The output is the raw weighted sum. If both inputs are at full amplitude with unity gains, the output can reach 2.0 peak.

CPU Cost

Minimal - two multiplications and one addition per sample. The Signal Mixer is one of the cheapest containers in the graph.

Connection Functions

The gainA and gainB ports support all standard connection functions (passthrough, add, multiply, subtract, replace, modulate). The source value comes from the upstream container’s controlOut. After the connection function is applied, the result is clamped to 0.0-2.0.
