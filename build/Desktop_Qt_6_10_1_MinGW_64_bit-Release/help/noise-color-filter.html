Noise Color Filter Documentation

Overview

The Noise Color Filter is a noise generator and shaper designed for breath and fricative effects. It can generate colored noise internally (white, pink, or brown) and shapes it through a configurable biquad filter to create different breath textures - from dark, breathy rumbles to bright, hissy fricatives.

This container serves dual purposes: it can generate noise internally or process external audio through its filter.

Category

Shaping (Orange) - Transforms spectral character

Core Concept

noise = generateNoise(noiseType)  // White, Pink, or Brown
output = biquadFilter(noise, color, filterQ, filterType)

The filter cutoff (color) determines the spectral character: - Low color (100 Hz) = dark, breathy rumble - Mid color (2000 Hz) = ‘SH’ territory - High color (8000 Hz) = ‘S’ territory, hissy

Ports

Input Ports

Port Data Type Default Description
audioIn Signal - External audio to filter (when not using internal noise)
color Control 2000.0 Filter cutoff frequency (100-12000 Hz)
filterQ Control 1.0 Filter resonance/sharpness (0.5-20.0)
pitchMultiplier Control 1.0 Scales the note frequency used as the filter center when followPitch is on. 1.0 = fundamental, 2.0 = octave above. Connect LFO or Envelope Engine for dynamic pitch tracking.

Output Ports

Port Data Type Description
noiseOut Signal Filtered noise output

Parameters

Parameter Type Default Range Description
color Float 2000.0 100.0-12000.0 Filter cutoff frequency in Hz. Ignored when followPitch is on.
filterQ Float 1.0 0.5-20.0 Filter Q factor (resonance). Higher values narrow the band — critical for pitched noise.
filterType Enum Highpass Lowpass, Highpass, Bandpass Filter mode. Use Bandpass with Follow Note Pitch for pitched noise effects.
noiseType Enum White White, Pink, Brown Internal noise generator type
followPitch Bool false on / off When on, the filter center frequency tracks the current note's fundamental frequency instead of using color. The color port modulation is also bypassed.
pitchMultiplier Float 1.0 0.5-8.0 Scales the note frequency before applying it to the filter. 1.0 = fundamental, 2.0 = octave above, 0.5 = octave below. Only active when followPitch is on.

Filter Types

Highpass (Default)

Passes frequencies above the cutoff, attenuates below. Default for vocal fricatives.

Use case: ‘S’ and ‘SH’ sounds, breath with air character.

Lowpass

Passes frequencies below the cutoff, attenuates above.

Use case: Dark breath, wind effects, muffled rumble.

Bandpass

Passes a band around the cutoff frequency, attenuates both above and below.

Use case: Focused breath textures, specific frequency emphasis.

Noise Types

White

Flat spectrum - equal energy per Hz. Bright, hissy character.

Pink

Energy proportional to 1/f. More natural, balanced character. Uses Paul Kellet’s algorithm.

Brown (Brownian)

Energy proportional to 1/f². Very low frequency emphasis, rumbling character. Generated via random walk.

Usage Patterns

Basic Breath Noise

Generate hissy breath for fricative consonants.

Noise Color Filter ──→ noiseOut ──→ [Mixer/Output]
     │
     └── filterType: Highpass, color: 4000, noiseType: White

Dark Breath

Low, breathy rumble for voiced breath.

Noise Color Filter ──→ noiseOut ──→ [Mixer/Output]
     │
     └── filterType: Lowpass, color: 500, noiseType: Brown

Animated Breath Color

Use LFO to sweep filter cutoff for evolving breath texture.

LFO ──→ color
          │
Noise Color Filter ──→ noiseOut ──→ [Output]

Pitched Noise (Flute / Edge-Tone Breath)

In flutes, recorders, shakuhachi, and similar edge-tone instruments, the turbulence at the embouchure is modulated by the bore's standing wave. The resulting noise is not spectrally flat — it has energy concentrated near the fundamental frequency and its harmonics. This preset recreates that effect.

Noise Color Filter ──→ noiseOut ──→ [Signal Mixer]
     │
     └── filterType: Bandpass
         followPitch: on
         pitchMultiplier: 1.0   (fundamental)
         filterQ: 5–15          (controls "how pitched" vs "how breathy")

Q as a timbral control:

  • Low Q (1–3): broad, breathy whisper with a pitch hint
  • Mid Q (5–10): clearly pitched breath, flute-like
  • High Q (15–20): narrow resonant band, almost tonal

Pitch Multiplier for harmonic colouring: Real instruments have pitched noise across multiple harmonics. Use two instances — one at pitchMultiplier: 1.0 and one at pitchMultiplier: 2.0 — mixed together for a richer, more acoustically accurate breath.

External Audio Processing

Process external audio instead of internal noise.

[External Audio] ──→ audioIn ──→ Noise Color Filter ──→ noiseOut
                                       │
                                       └── useInternal: false

Comparison with Other Containers

Container Domain Purpose
Noise Color Filter Signal Generates and shapes noise for breath
Rolloff Processor Spectrum Power-law harmonic attenuation
Formant Body Signal Resonant formant filtering

Noise Color Filter generates its own noise source and shapes it for breath/turbulence, while the other shapers process existing harmonic or signal content.

Technical Notes

Biquad Filter

Uses standard biquad filter implementation (Direct Form I) with coefficients from the Audio EQ Cookbook by Robert Bristow-Johnson.

output = b0×input + b1×x[n-1] + b2×x[n-2] - a1×y[n-1] - a2×y[n-2]

Pink Noise Algorithm

Implements Paul Kellet’s pink noise algorithm using a bank of first-order filters to achieve approximately -3 dB/octave rolloff.

Brown Noise Algorithm

Uses random walk with leaky integration:

brownLast = (brownLast + 0.02 × white) / 1.02

Phase Independence

The internal noise generator uses time-seeded random number generation, so each instance produces different noise patterns.
