Harmonic Generator Documentation

Overview

The Harmonic Generator creates the fundamental harmonic spectrum — the “bones” of sound in Kala. It generates rich harmonic content using additive synthesis, providing the raw material that other containers shape and transform into final sound.

When PADsynth mode is enabled, the generator additionally pre-computes a large wavetable using the PADsynth algorithm: each harmonic is spread across a frequency-domain band using a configurable profile, then the whole spectrum is inverse-FFT’d into a single-cycle wavetable. This gives sounds a characteristic lush, organic, slightly chorused quality that is difficult to achieve with pure additive synthesis alone. The PADsynth wavetable is passed automatically to a connected Spectrum to Signal container for playback.

DNA Presets

0: Vocal (Bright)

  • Harmonic profile: 1.2 power rolloff
  • Boosts: Harmonics 3–5 and 8–12 boosted 1.4×
  • Character: Bright, present, forward
  • Use for: Lead vocals, bright solo instruments

1: Vocal (Warm)

  • Harmonic profile: 1.6 power rolloff
  • Boosts: Harmonics 2–4 boosted 1.5×
  • Character: Warm, mellow, recessed
  • Use for: Backing vocals, warm pads, male vocals

2: Brass (Trumpet-like)

  • Harmonic profile: 1.5 power rolloff
  • Boosts: Odd harmonics boosted 1.3×
  • Character: Bright, brassy, buzzy
  • Use for: Trumpets, trombones, brass sections

3: Reed (Clarinet-like)

  • Harmonic profile: Dominant odd harmonics (1/n^1.4), very weak even harmonics (0.1/n^2.0)
  • Character: Hollow, woody, nasal
  • Use for: Clarinets, oboes, reed instruments

4: String (Violin-like)

  • Harmonic profile: 1.3 power rolloff
  • Boosts: Harmonics 5–10 boosted 1.2×
  • Character: Rich, present mid-range
  • Use for: Violins, cellos, string sections

5: Pure Sine (Test)

  • Harmonic profile: Only fundamental present
  • Character: Clean, toneless
  • Use for: Testing, reference tone

Custom DNA

When dnaSelect = −1, the DNA Editor dialog lets you define per-harmonic amplitudes directly. Built-in custom patterns include:

  • All Harmonics — standard harmonic series
  • Odd Only — clarinet-like (1, 3, 5, 7…)
  • Even Only — hollow timbre (2, 4, 6, 8…)
  • Odd Dominant — strong odd harmonics, weak even
  • Fundamental+ — near-sine with very weak overtones
  • Octaves Only — organ-like (1, 2, 4, 8, 16…)
  • Fibonacci Sequence — harmonics at Fibonacci numbers
  • Prime Numbers — harmonics at prime numbers

Digit Formula DNA

Digit Formula is a DNA mode that derives harmonic amplitudes from the decimal digits of a number. Each fractional digit (0–9) maps to an amplitude of digit / 9.0, giving a natural 0–1 range without any explicit scaling. The result is a harmonic spectrum whose shape is defined by the mathematical structure of the number — periodic for rationals, non-repeating for irrationals.

Select Digit Formula from the Sequence dropdown in the DNA Editor to access this mode.

Concepts

Rational numbers — periodic spectra

Any ratio of two integers produces a repeating decimal. For example, 1/7 = 0.142857142857142857… — the six-digit period 142857 repeats forever. Across 64 harmonics this period cycles ~10 times, creating a periodic amplitude lattice: harmonics 1, 7, 13, 19… all share the same amplitude; harmonics 2, 8, 14, 20… share another. The periodicity is audibly present as a characteristic timbral grain.

Fractions with prime denominators produce the longest periods. The period length of 1/p divides p−1, so:

Formula Period Character
1/7 6 digits Short, strongly periodic
1/13 6 digits Different six-note pattern
1/17 16 digits Two-octave-equivalent cycle
1/97 96 digits No repetition within 64 harmonics — sounds irrational

Rational digit extraction uses exact integer arithmetic (__int128 long division) — there is no floating-point precision limit. The window can scroll to digit 9999 and the digits are always correct.

Irrational constants — non-repeating spectra

The built-in constants (π, e, φ, √2, √3, √5) have 120 hardcoded fractional digits each. Every window position gives a genuinely unique harmonic distribution with no repeating structure.

Constant Digits available Character
π (Pi) 120 Irregular, unpredictable
e (Euler) 120 Similar irregularity, different pattern
φ (Golden Ratio) 120 Related to Fibonacci — subtle near-periodicity
√2, √3, √5 120 Algebraic irrationals, dense and varied

The window

The Start digit spinbox in the DNA Editor selects where in the fractional expansion to begin reading. With 64 harmonics and start = 0, digits 0–63 are used. At start = 64, digits 64–127 are used — a completely different spectrum from the same number. Sliding the window is how you explore the mathematical landscape of a constant or ratio.

With PADsynth

Combining Digit Formula DNA with PADsynth is particularly effective. The digit period becomes a harmonic skeleton; PADsynth spreads each harmonic into a frequency-domain cloud. For 1/7, the six period classes become six overlapping spectral blooms at different amplitudes — the periodicity remains audible as a structural grain, but each harmonic has organic width and warmth. Different window positions shift which class gets which amplitude, morphing the instrument while keeping its mathematical DNA.

Window Offset as a Performance Parameter

The digitWindowOffset input port allows the window to be animated during playback by a control signal. As the offset shifts, new digit positions are read and harmonic amplitudes update — the timbre morphs continuously through the mathematical landscape of the formula while preserving its fundamental DNA character.

Setup:

  1. Set DNA to Digit Formula in the DNA Editor and click OK
  2. Add an Envelope Engine (set to Score Curve mode)
  3. Connect its output to the digitWindowOffset port of the Harmonic Generator
  4. Set the connection weight to control the sweep range (e.g. weight = 50 → offset sweeps 0 to 50 over the curve)
  5. Draw a curve in the score for that note
[Score Curve] → [Envelope Engine (score curve)] → digitWindowOffset → [Harmonic Generator]
                                                   weight: 50
                                                   → offset sweeps 0–50 over note duration

The Window Offset spinbox in the inspector sets the base offset — the static starting position used when no port is connected, or as an additive offset when a port is connected.

With 1/7 (period 6) and a sweep of 6 digits, the timbre makes exactly one full period rotation and returns to its starting character — a closed timbral arc. With π and a sweep of 50 digits, the timbre traverses 50 unique harmonic configurations — an open, non-repeating journey.

Persistence

The precomputed digit string (up to 1000 digits) is stored in both .kala project files and .sounit files, so the full window range is always available after loading without any recomputation.

Purity Control

Blends between the DNA preset and a flat spectrum:

amplitude[h] = dna[h] × (1 − purity) + (1/h) × purity
  • 0.0: Pure DNA preset — full character
  • 1.0: Flat 1/h spectrum — even harmonic distribution
  • Intermediate: Weighted blend

Drift Control

Adds organic variation through per-harmonic frequency modulation:

driftOffset[h] += drift × 0.0001 × sin(phase × 0.023 + h × 1.3)
harmonicPhase += driftOffset[h] × 2π
  • 0.0: Perfectly stable harmonics
  • Small values (0.01–0.03): Subtle organic beating
  • Higher values: Pronounced detuning, ensemble-like texture

PADsynth Mode

PADsynth is an algorithm originally from ZynAddSubFX (Paul Nasca). Instead of summing sine waves at integer harmonics, it spreads each harmonic’s energy across a frequency band in the FFT domain before converting to a time-domain wavetable. The result is a sound that has a slightly blurred, lush quality — harmonics are no longer infinitely thin spectral lines but have a gentle width, which creates natural-sounding beating and warmth.

How It Works

  1. Start from the DNA harmonic amplitudes — the same amplitudes used by normal additive synthesis (including purity blend)

  2. For each harmonic, compute a bandwidth in Hz:

    bw_hz = (2^(padBandwidth/1200) − 1) × fundamental × harmonicNumber^padBandwidthScale

  3. Spread the harmonic’s amplitude across neighboring FFT bins using the chosen profile shape

  4. Assign random phases to each bin (fixed seed 42 — deterministic per build)

  5. Inverse FFT the entire frequency-domain buffer → time-domain wavetable

  6. Normalize the wavetable to peak ±1.0

  7. Pass the wavetable to Spectrum to Signal for playback at any pitch

The wavetable is generated once at graph build time at a reference pitch of 131 Hz (≈ C3). Spectrum to Signal plays it back at the correct pitch by adjusting its read speed.

padBandwidth — Harmonic Spread

Controls the width of each harmonic’s energy spread in cents. One cent = 1/100 of a semitone.

padBandwidth Effect
1–5 cents Very narrow — barely different from additive synthesis
10–20 cents Subtle warmth, slight beating
40 (default) Noticeable bloom — classic PADsynth character
80–100 cents Wide spread — lush, heavily chorused
150–200 cents Very wide — almost noisy, harmonics merge together

padBandwidthScale — Harmonic Growth

Controls how bandwidth grows with harmonic number:

effective_bw = bw_hz × harmonicNumber^padBandwidthScale
padBandwidthScale Effect
0.0 All harmonics have the same absolute bandwidth (Hz)
1.0 (default) Bandwidth grows linearly with harmonic number — physically realistic (constant cents per harmonic)
2.0 Bandwidth grows quadratically — upper harmonics become very wide and diffuse

At scale = 1.0, each harmonic has the same bandwidth in cents, which matches the perceptual model of how musical instruments spread.

padProfileShape — Spread Shape

The profile function determines how each harmonic’s amplitude falls off as you move away from its center frequency.

0: Gaussian (default)

profile(x) = exp(−x² × 2.7726)    [−6 dB at ±1 bw]

Smooth, continuous rolloff — no hard edges. The most natural-sounding profile; creates smooth beating. Tails extend beyond the nominal bandwidth (spread ≈ 3× bw to capture the Gaussian tails).

Character: Warm, smooth, organic. The classic PADsynth sound.

1: Raised Cosine

profile(x) = 0.5 × (1 + cos(π × x))    for |x| ≤ 1, else 0

Cosine-shaped bell that goes cleanly to zero at the bandwidth boundary. Slightly more defined than Gaussian — the harmonics remain more distinct.

Character: Warm but more focused than Gaussian. Good balance between spread and definition.

2: Square

profile(x) = 1    for |x| ≤ 1, else 0

Flat top within the bandwidth, hard cutoff outside. Creates a more buzzy, less smooth spread — the bins within the bandwidth all have equal amplitude.

Character: Rougher texture, more artificial-sounding. Good for metallic or electronic timbres where a more uniform spread is desirable.

Connecting PADsynth to SpectrumToSignal

PADsynth output flows through the normal Spectrum port connection. When Spectrum to Signal detects an incoming PADsynth wavetable from the Harmonic Generator, it switches automatically to wavetable playback mode:

[Harmonic Generator] --spectrum--> [Spectrum to Signal] --> signalOut
  padEnabled: true                  (auto-detects PADsynth
  padBandwidth: 40                   wavetable and uses it)

No special configuration is needed on the Spectrum to Signal side — wavetable detection is automatic.

The wavetable is also propagated through Spectrum Blender connections (spectrumA input), so you can blend two PADsynth spectra:

[HG 1 - Vocal, PADsynth] --spectrum--> spectrumA -+
                                                    |--> [Spectrum Blender] --spectrum--> [S2S]
[HG 2 - String, PADsynth] --spectrum--> spectrumB -+

PADsynth Workflow

  1. Set the DNA preset and harmonic count as usual
  2. Open the inspector for the Harmonic Generator
  3. Enable PADsynth (check padEnabled)
  4. Adjust padBandwidth — start around 30–60 cents
  5. Choose padProfileShape (Gaussian is a good default)
  6. Leave padBandwidthScale at 1.0 unless you want unusual harmonic spread behavior
  7. padFftSize of 262144 (256k) gives high quality; reduce to 65536 for faster builds
  8. Rebuild the graph (PADsynth generates on graph build, not per-sample)

PADsynth CPU Cost

PADsynth generation happens once at graph build time — it is not a per-sample cost. A 262144-sample FFT with kiss_fft takes a few tens of milliseconds, transparent to the user. At render time, playback is simply a table lookup in SpectrumToSignal — no additional overhead compared to normal synthesis.

The padFftSize setting trades quality for build time:

padFftSize Quality Build time (approx.)
65536 (64k) Good Very fast
262144 (256k, default) High Fast
524288 (512k) Very high Moderate
1048576 (1M) Highest Slow

Standard Additive Synthesis

[Harmonic Generator] --spectrum--> [Rolloff Processor] --spectrum--> [Spectrum to Signal] --> Output
  dnaSelect: 0 (Vocal Bright)
  numHarmonics: 64

PADsynth Lush Pad

[Harmonic Generator] --spectrum--> [Spectrum to Signal] --> Output
  dnaSelect: 1 (Vocal Warm)
  numHarmonics: 32
  padEnabled: true
  padBandwidth: 60
  padProfileShape: 0 (Gaussian)

PADsynth with Rolloff Shaping

[Harmonic Generator] --spectrum--> [Rolloff Processor] --spectrum--> [Spectrum to Signal] --> Output
  padEnabled: true
  padBandwidth: 40

The Rolloff Processor shapes the spectrum before PADsynth sees the amplitudes — so rolloff affects the spread width proportionally.

Digit Formula — Static Timbre

[Harmonic Generator] --spectrum--> [Spectrum to Signal] --> Output
  dnaSelect: -1 (Custom, Digit Formula)
  formula: 1 ÷ 97   (96-digit period, no repetition within 64 harmonics)
  digitWindowOffset: 0

Digit Formula + PADsynth

[Harmonic Generator] --spectrum--> [Spectrum to Signal] --> Output
  formula: 1 ÷ 7   (period 142857)
  padEnabled: true
  padBandwidth: 20    (narrow → clear periodic grain)
  padProfileShape: 0  (Gaussian)

Try increasing padBandwidth to 80+ to turn the six harmonic classes into broad overlapping clouds.

Digit Formula — Animated Window

[Envelope Engine (score curve)] --controlOut--> digitWindowOffset --> [Harmonic Generator]
                                  weight: 96                              formula: 1 ÷ 97
                                                                          padEnabled: true

With weight=96 and a linear score curve, the window sweeps exactly one full period of 1/97 over the note duration — the timbre makes a complete closed rotation through all 96 harmonic configurations and returns to its starting character.

Drift + PADsynth

Both drift and PADsynth add organic movement, but through different mechanisms: - Drift adds slow per-harmonic frequency variation at runtime - PADsynth bakes harmonic spread into the wavetable at build time

Using both together is valid and produces very lush results, though at some point they become redundant. Try PADsynth alone first (drift = 0) to hear its characteristic texture before adding drift.

Purity Modulation with PADsynth

[Drift Engine] --> controlOut --> purity --> [Harmonic Generator] --spectrum--> [S2S]
                                              padEnabled: true

Note: Changing purity during playback does not regenerate the PADsynth wavetable — the wavetable is fixed at graph build time. Purity modulation affects the normal spectrum output (used by the Rolloff Processor, Spectrum Blender, etc.) but not the PADsynth playback.

Troubleshooting

No Sound Output

  1. Ensure a Spectrum to Signal is connected downstream
  2. Check that numHarmonics > 0
  3. Verify the note pitch is a valid frequency

PADsynth Has No Effect

  1. Confirm padEnabled is set to 1 in the inspector
  2. Rebuild the graph — PADsynth generates at build time, not dynamically
  3. Verify Spectrum to Signal is directly connected (wavetable propagation requires the spectrum connection)

PADsynth Sounds Identical to Normal Synthesis

  1. Increase padBandwidth — very small values (< 5 cents) are nearly imperceptible
  2. Try a large FFT size (262144 or higher) for finer frequency resolution

PADsynth Sounds Too Blurry / Noisy

  1. Decrease padBandwidth — reduce toward 10–20 cents
  2. Decrease padBandwidthScale toward 0 — keeps upper harmonics from becoming too wide
  3. Try Raised Cosine profile (less tail energy than Gaussian)

Static or Boring Sound

  1. Try different DNA presets
  2. Add Rolloff Processor downstream for brightness control
  3. Enable PADsynth for organic texture
  4. Add small drift (0.01–0.03) for additional movement

CPU Performance Issues

  1. Reduce numHarmonics (16–32 is a good balance)
  2. PADsynth itself adds no per-sample cost — only the graph rebuild is affected by padFftSize
  3. Reduce padFftSize to 65536 if build time is a concern

Technical Notes

Harmonic Amplitude Formula (Normal Mode)

Power-law rolloff: amplitude[h] = 1 / harmonicNumber^rolloffPower

PADsynth Bandwidth Formula

bw_hz = (2^(padBandwidth/1200) − 1) × fundamentalHz × harmonicNumber^padBandwidthScale

For 40 cents at 131 Hz, harmonic 1: bw_hz ≈ 0.0234 × 131 × 1 ≈ 3.06 Hz

Deterministic Random Phases

The PADsynth implementation uses a fixed LCG seed (42) to assign random phases to each frequency bin. This means the wavetable is always the same for a given set of parameters and DNA preset — the result is reproducible across renders.

Normalization

All harmonic amplitudes are normalized so their sum equals 1.0. The PADsynth wavetable is independently normalized to peak ±1.0 after the inverse FFT.
