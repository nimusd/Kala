Easing Applicator Documentation

Overview

The Easing Applicator is a stateless mathematical interpolator that applies easing functions to blend between two values based on a progress input. Unlike a transition engine that manages time internally, the Easing Applicator is a pure function: given a start value, end value, and progress (0-1), it outputs an eased interpolation.

This design makes it highly composable - you control the progress externally (from an Envelope Engine, LFO, Physics System, or any other source), and the Easing Applicator shapes how that progress translates to the output value.

Category

Modifiers (Green) - Shapes how parameter changes are interpolated

Core Concept

easedValue = startValue + (endValue - startValue) × easingFunction(progress)

The easing function transforms the linear progress (0→1) into a curved progress, creating acceleration, deceleration, overshoot, bounce, or other motion characteristics.

Ports

Usage Patterns

Basic Value Interpolation

The simplest use: interpolate between two fixed values based on external progress.

LFO (0-1 output) ──→ progress
                              ├──→ Easing Applicator ──→ easedValue
Constant (0.2) ──→ startValue
Constant (0.8) ──→ endValue

Result: The LFO’s linear 0-1 ramp becomes an eased transition between 0.2 and 0.8.

Envelope-Driven Parameter Morphing

Use an envelope to drive progress while easing shapes the motion.

Envelope Engine ──→ progress
                              ├──→ Easing Applicator ──→ [Filter Cutoff]
Note Attack (500 Hz) ──→ startValue
Note Sustain (2000 Hz) ──→ endValue

Result: Filter cutoff follows envelope timing but with eased (not linear) motion.

Pitch Glide / Portamento

Connect pitch values and use progress from a timing source.

Previous Note Pitch ──→ startValue
                                     ├──→ Easing Applicator ──→ Spectrum to Signal (pitch)
Current Note Pitch ──→ endValue
Envelope (glide) ──→ progress

Result: Pitch slides between notes with customizable acceleration curve.

Physics-Controlled Easing

Let a Physics System provide the progress for organic motion.

Physics System ──→ progress
                              ├──→ Easing Applicator ──→ [Parameter]

Result: Physics-based acceleration combined with easing character.

LFO-Shaped Modulation

Apply easing to reshape an LFO’s waveform.

LFO (sawtooth) ──→ progress
                              ├──→ Easing Applicator (Bounce Out) ──→ [Vibrato depth]

Result: Each LFO cycle bounces rather than ramps linearly.

Important Behaviors

Overshoot Handling

Back and Elastic easings can produce output values outside the startValue-endValue range:

  • Back In: Output briefly goes below startValue
  • Back Out: Output briefly goes above endValue
  • Elastic: Output oscillates past both boundaries

If your target parameter has hard limits, either: 1. Choose non-overshooting easings (IDs 0-18, 25-27) 2. Clamp the output in your signal chain 3. Use smaller overshoot/amplitude parameters

Progress Clamping

The progress input is automatically clamped to 0.0-1.0. Values outside this range are brought to the boundaries before the easing function is applied.

Parameter Update Timing

Easing parameters (easingSelect, overshoot, amplitude, etc.) are read each processing cycle. You can modulate these in real-time for evolving easing behavior.

Stateless Operation

The Easing Applicator has no internal state or memory. Given the same inputs, it always produces the same output. This makes it: - Predictable: Output depends only on current inputs - Composable: Chain multiple easings or use in parallel - Efficient: No internal buffers or state management

Comparison with Other Modifiers

Container Purpose State
Easing Applicator Shape interpolation between values Stateless
Physics System Emergent motion from mass/spring Stateful (momentum)
Envelope Engine Time-based value automation Stateful (position in envelope)
LFO Periodic oscillation Stateful (phase)
Drift Engine Random wandering Stateful (current drift)

The Easing Applicator is unique in being purely mathematical - it transforms input to output with no memory of previous values. This makes it ideal for reshaping the output of stateful containers.

Technical Notes

Implementation

The Easing Applicator wraps the Easing class, which implements all 29 easing functions. The core calculation:

double easedProgress = easing.calculate(progress);  // Apply easing to 0-1 progress
return startValue + (endValue - startValue) * easedProgress;  // Interpolate

Performance

  • Pure mathematical operations (no allocations, no I/O)
  • Single function call per sample
  • Negligible CPU impact even with many instances

Serialization

Easing configurations are saved/loaded as part of the Sounit graph. The Easing class supports JSON serialization including all custom parameters.
